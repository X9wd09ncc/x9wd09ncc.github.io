虚拟主机发现

在网站首页邮箱处找到一个 hostname

目录扫描

访问 /robots.txt

漏洞利用

本地文件包含 (LFI)

访问 /test.php，有一个按钮

浏览器访问并点击按钮，注意到参数 view 后面跟的是目录，猜测存在目录遍历漏洞

用 base64 去解析，返回了文件的编码结果

解码后发现限制条件：

不能包含 ../..（防止路径向上跳转）
必须包含 /var/www/html/development_testing（强制从该目录开始）

虽然不能有 ../..，但是 .. 本身是允许的，因此可以用双斜杠 ..//.. 绕过

绕过过滤

尝试目录遍历，成功返回了 /etc/passwd 的内容

日志投毒

尝试遍历阿帕奇的日志文件，也是成功

修改 User-Agent 参数来污染日志文件

然后再执行 cmd 命令，将结果输出到 log.txt

查看日志文件，在最后找到了 id 的执行结果

反弹 Shell

那就直接反弹 shell

成功反弹

权限提升

Cron 定时任务

查看用户 archangel 的可执行文件，有一个 helloworld.sh

而且 www-data 用户有对该文件的修改权限

查看定时任务，发现该文件每分钟会以 archangel 权限执行一次

修改该文件，将反弹命令写进去

再开一个窗口进行监听，等一会就成功拿到权限

SUID 提权

在 secret 找到第二个 flag，同时还有一个 SUID 二进制文件

将该文件下载到本地分析，该文件会执行 cp 命令

PATH 劫持

在 /tmp 目录下创建一个假的 cp 命令，并赋予 777 权限

修改 PATH 环境变量，让系统优先使用我们伪造的 cp

执行 ./backup，成功提权

总结

知识点

  • 本地文件包含漏洞（LFI）
  • 日志投毒（Log Poisoning）
  • PHP 伪协议绕过
  • Cron 定时任务提权
  • SUID 二进制文件
  • PATH 劫持提权
