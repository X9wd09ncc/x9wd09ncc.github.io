Library

题目描述：boot2root 机器适用于 FIT 和 bsides 危地马拉 CTF

信息收集

端口扫描

序号 Host地址 端口 协议 Uri地址 标题 Banner 状态码
1 10.145.170.29 80 HTTP http://10.145.170.29:80 欢迎来到博客 - 图书馆机器 Apache/2.4.18 (Ubuntu),Apache-Web-Server,Apache-HTTP-Server/2.4.18 200
2 10.145.170.29 22 SSH 10.145.170.29:22 OpenSSH 7.2p2 Ubuntu 4ubuntu2.8 0

目录扫描

访问 80 端口，是一个博客页面，其他地方点击无响应，出现了一个用户名：meliodas

扫描目录得到 robots.txt

访问 /robots.txt

漏洞利用

SSH 暴力破解

端口有开放 22 端口，80 端口发现用户名，/robots.txt 提示 rockyou 字典，于是尝试用 hydra 破解 ssh 密码

成功破解得到密码 iloveyou1

SSH 登录

使用用户名 meliodas 和密码 iloveyou1 登录 ssh 服务

权限提升

Sudo 提权

使用 sudo -l 发现可以无密码 root 执行命令 /usr/bin/python* /home/meliodas/bak.py

脚本分析

查看 bak.py，只是一个备份脚本，把 /var/www/html 打包成 /var/backups/website.zip

并没有可以提权的点

尝试修改 bak.py，但是没有权限

恶意模块提权

后面注意到 bak.py 引用了 zipfile，于是可以构造一个恶意的 zipfile.py，在运行 bak.py 的时候他就会优先调用同目录下的 zipfile.py，从而达到提权的目的

总结

知识点

  • Hydra SSH 暴力破解
  • Python 模块劫持提权
  • Sudo 提权：利用可执行脚本的依赖模块
