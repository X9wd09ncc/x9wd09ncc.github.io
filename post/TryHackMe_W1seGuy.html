已知明文 XOR 攻击漏洞渗透测试报告

一、漏洞概述

项目 内容
漏洞名称 已知明文攻击 (Known-Plaintext Attack) / XOR 密钥泄露
漏洞编号 CWE-327 (使用破损的加密算法)
目标系统 10.49.149.157:1337 (TCP)
发现日期 2026-05-25
风险等级 高危 (High)
CVSS 评分 9.1 (CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:N)
漏洞类型 加密实现缺陷 / 信息泄露

二、漏洞描述

目标系统在 TCP 端口 1337 上运行一个 Python 服务，声称通过 XOR 加密保护 flag。服务每次连接时：

  1. 生成一个 5 字符随机密钥（从 ascii_letters + digits 中选取）
  2. 使用密钥对已知字符串 THM{thisisafakeflag} 进行重复 XOR 加密
  3. 将密文以十六进制形式发送给客户端
  4. 要求客户端猜测密钥，答对则返回真正的 flag

核心问题：

  • 攻击者已知被加密的明文内容（THM{thisisafakeflag}）
  • 密钥长度仅 5 字符，且通过已知明文的前 4 字节（THM{）和后 1 字节（}）即可完全恢复
  • 恢复密钥后，可解密服务端实际加密的 40 字节真正 flag

三、源代码分析

3.1 服务逻辑

def setup(server, key):
    flag = 'THM{thisisafakeflag}' 
    xored = ""
    for i in range(0, len(flag)):
        xored += chr(ord(flag[i]) ^ ord(key[i % len(key)]))
    hex_encoded = xored.encode().hex()
    return hex_encoded

def start(server):
    res = ''.join(random.choices(string.ascii_letters + string.digits, k=5))
    key = str(res)
    hex_encoded = setup(server, key)
    send_message(server, "This XOR encoded text has flag 1: " + hex_encoded + "\n")
    send_message(server, "What is the encryption key? ")
    key_answer = server.recv(4096).decode().strip()
    if key_answer == key:
        send_message(server, "Congrats! That is the correct key! Here is flag 2: " + flag + "\n")

3.2 关键缺陷

缺陷 说明
密钥过短 仅 5 字符，密钥空间仅 62^5 ≈ 9.1 亿，可暴力破解
密钥重复使用 XOR 密钥循环应用于整个明文，导致周期性模式
已知明文 攻击者确切知道被加密的字符串是 THM{thisisafakeflag}
结构可预测 Flag 格式固定为 THM{...}，前 4 字节和后 1 字节确定
无随机填充 无 IV、无盐值，每次加密完全确定性

四、攻击原理

4.1 XOR 加密的数学性质

XOR 加密满足：密文[i] = 明文[i] ⊕ 密钥[i % len(key)]

因此：密钥[i % len(key)] = 密文[i] ⊕ 明文[i]

4.2 密钥恢复攻击

已知：

  • 密文（服务器发送的 hex 字符串）
  • 明文前 4 字节：T H M {
  • 明文最后 1 字节：}

密钥长度 = 5，因此：

  • key[0] = ciphertext[0] ⊕ 'T'
  • key[1] = ciphertext[1] ⊕ 'H'
  • key[2] = ciphertext[2] ⊕ 'M'
  • key[3] = ciphertext[3] ⊕ '{'
  • key[4] = ciphertext[-1] ⊕ '}'（密文长度 40，40 % 5 = 0，对应 key[0]… 不对，40-1=39，39 % 5 = 4，对应 key[4] ✓）

4.3 Flag 解密

恢复密钥后，对密文全部 40 字节进行 XOR 解密：

plaintext[i] = ciphertext[i] ⊕ key[i % 5]

五、攻击复现

5.1 连接服务器

echo "" | nc -w 5 10.49.149.157 1337

服务器响应：

This XOR encoded text has flag 1: 073e1818406217390d44160e212244274236085312182750513f3a2c0b6521022c5345210e1a114d
What is the encryption key?

5.2 恢复密钥

cipher = bytes.fromhex("073e1818406217390d44160e212244274236085312182750513f3a2c0b6521022c5345210e1a114d")
key = ''
key += chr(cipher[0] ^ ord('T'))  # 'h'
key += chr(cipher[1] ^ ord('H'))  # 'q'
key += chr(cipher[2] ^ ord('M'))  # 'h'
key += chr(cipher[3] ^ ord('{'))  # 'I'
key += chr(cipher[-1] ^ ord('}')) # 'z'
# key = 'hqhIz'

5.3 解密 Flag 1

plaintext = ''.join(chr(cipher[i] ^ ord(key[i % 5])) for i in range(len(cipher)))
# 结果: THM{p1alntExtAtt4ckcAnr3alLyhUrty0urxOr}

5.4 获取 Flag 2

echo "hqhIz" | nc -w 5 10.49.149.157 1337

服务器响应：

Congrats! That is the correct key! Here is flag 2: THM{BrUt3_ForC1nG_XOR_cAn_B3_FuN_nO?}

六、完整利用脚本

import socket
import string

def exploit(host, port):
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.settimeout(10)
    s.connect((host, port))

    # Read all data from server
    all_data = b''
    while True:
        try:
            chunk = s.recv(4096)
            if not chunk: break
            all_data += chunk
        except socket.timeout:
            break

    # Extract hex-encoded ciphertext
    import re
    match = re.search(r'flag 1: ([0-9a-fA-F]+)', all_data.decode())
    hex_str = match.group(1)
    cipher = bytes.fromhex(hex_str)

    # Recover key from known plaintext (THM{ prefix + } suffix)
    key = ''
    prefix = 'THM{'
    for i in range(4):
        key += chr(cipher[i] ^ ord(prefix[i]))
    key += chr(cipher[-1] ^ ord('}'))

    print(f"Recovered key: {key}")

    # Decrypt full flag 1
    plaintext = ''.join(chr(cipher[i] ^ ord(key[i % 5])) for i in range(len(cipher)))
    print(f"Flag 1: {plaintext}")

    # Send key to get flag 2
    s.send((key + '\n').encode())
    import time; time.sleep(0.5)
    response = b''
    try:
        while True:
            chunk = s.recv(4096)
            if not chunk: break
            response += chunk
    except socket.timeout: pass
    print(f"Flag 2: {response.decode()}")
    s.close()

exploit('10.49.149.157', 1337)

七、影响评估

影响维度 评估
机密性 完全破坏 — 攻击者可恢复所有加密的 flag
完整性 无直接影响
可用性 无直接影响
利用难度 极低 — 无需认证，连接即发送密文，数学上可确定性恢复密钥
攻击面 任何可连接 TCP 1337 端口的网络位置

八、修复建议

8.1 短期修复（立即实施）

  1. 使用密码学安全的加密算法

    • 替换 XOR 为 AES-GCM、ChaCha20-Poly1305 等认证加密算法
    • 使用 cryptography 库而非手写加密

  2. 密钥长度至少 256 位

    • 当前 5 字符（≈ 30 位熵）远远不够
    • 使用 os.urandom(32) 生成 256 位随机密钥

8.2 中期改进

  1. 使用随机 IV/Nonce

    • 每次加密使用不同的初始化向量
    • 防止相同明文产生相同密文

  2. 不要暴露已知明文

    • 不要在发送密文的同时暴露被加密的内容
    • 使用认证加密确保密文完整性

8.3 长期架构改进

  1. 遵循 Kerckhoffs 原则

    • 加密系统的安全性应仅依赖于密钥保密，而非算法保密
    • 即使攻击者知道算法和明文结构，也无法在不知道密钥的情况下解密

  2. 使用标准加密库

    • 永远不要自己实现加密算法
    • 使用经过同行评审的标准库（如 cryptography, libsodium）
