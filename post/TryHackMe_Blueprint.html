目录扫描

访问 8080 端口，是一个文件目录，依次点击来到 /oscommerce-2.3.4/catalog，是一个网页

根据路径知道使用了 oscommerce-2.3.4

得到以下目录：

  • /admin (Status: 301)
  • /download (Status: 401)
  • /ext (Status: 301)
  • /images (Status: 301)
  • /includes (Status: 301)
  • /index.php (Status: 200)
  • /install (Status: 301)
  • /pub (Status: 301)

漏洞搜索

目录扫描的同时用 searchsploit 搜索相关漏洞

可以利用的有文件上传和远程代码执行

漏洞利用

安装 osCommerce

访问扫描出来的地址，在 /oscommerce-2.3.4/catalog/install 发现一个安装页面

点击开始按钮安装

这里数据库服务器地址默认是 localhost，但是我修改为靶场地址后会出错，因此需要修改 hosts 文件

用户名用 root，无密码，数据库名则是 osCommerce

点击继续按钮

点击继续按钮

其他信息乱填，管理员账号密码均为 admin

文件上传获取 Shell

访问 /admin，可以登录

下载之前搜索到的任意文件上传利用脚本文件 43191.py，上传单文件 PHP shell

访问 /oscommerce-2.3.4/catalog/admin/shell.php

执行 whoami 得知是最高权限，进入 Desktop 查看 root.txt.txt

权限提升

获取 NTLM Hash

第一个题目要求我们获取 lab 用户的 NTLM hash 值并解密

msf 里面有一个 hashdump 命令可以获取该值

先用 msfvenom 生成反弹 payload，然后用 43191.py 将 shell.exe 上传

端口转发

我这里比较特殊，我是 win11 用 openvpn 连接的靶场，所以 kali 虚拟机能访问靶场地址，但是无法接受靶场的反弹，因此需要进行端口转发

Metasploit 监听

进入 msfconsole，使用 exploit/multi/handler 模块，设置 port 以及 ip，然后 run 开启监听

再去之前的 shell 处运行 shell.exe

成功反弹，执行 hashdump 获取 Lab 用户的哈希值

解密 Hash

在 https://hashes.com/en/decrypt/hash 网站进行解密

总结

知识点

  • osCommerce 2.3.4 漏洞利用
  • 文件上传漏洞
  • Metasploit hashdump 获取 NTLM Hash
  • 端口转发技巧
