TryHackMe Juicy Details - 日志分析报告

概述

本报告分析 logs_1618139984797 文件夹中的三份日志文件：access.log（Web 访问日志）、auth.log（系统认证日志）和 vsftpd.log（FTP 服务日志）。所有攻击流量均来自攻击者 IP：192.168.10.5，目标服务器为 192.168.10.4。攻击发生在 2021年4月11日 08:29 - 09:43 UTC。

攻击时间线

时间 (UTC) 阶段 描述
08:29 - 09:08 侦察 Nmap 扫描、FTP 连接、匿名登录
09:08 - 09:13 初始访问 浏览 Web 应用、注册账户、登录
09:13 - 09:15 信息收集 浏览产品评论，抓取电子邮件地址
09:16 暴力破解 Hydra 对 /rest/user/login 发起暴力破解
09:16:31 暴力破解成功 Hydra 成功登录其他用户账户
09:20 - 09:29 横向移动 使用被盗账户浏览、尝试 SQL 注入
09:29 - 09:31 SQL 注入 sqlmap 扫描 + 手动 UNION 注入提取用户数据
09:34 - 09:36 文件发现与下载 feroxbuster 发现 /ftp、通过 FTP 下载文件
09:35 - 09:41 SSH 暴力破解 对 www-data 用户发起 SSH 暴力破解
09:41:19 SSH 登录成功 获取 shell 访问权限
09:42 - 09:43 权限提升尝试 sudo 和 su 命令活动

问题详细分析

Q1: 攻击者使用了哪些工具？

通过分析 access.log 中的 User-Agent 字段，识别出以下五种工具：

工具 出现次数 用途
Hydra 288 Web 登录暴力破解 (/rest/user/login)
sqlmap 78 SQL 注入自动化扫描与利用 (/rest/products/search)
Nmap 10 网络侦察与扫描
feroxbuster 9 目录/文件暴力枚举
curl 1 手动 HTTP 请求（用于确认 SQL 注入结果）

答案: Hydra, sqlmap, Nmap, feroxbuster, curl

证据:

  • Nmap: "Mozilla/5.0 (compatible; Nmap Scripting Engine; https://nmap.org/book/nse.html)" — access.log 行 2-11
  • Hydra: "Mozilla/5.0 (Hydra)" — access.log 行 208-495
  • sqlmap: "sqlmap/1.5.2#stable (http://sqlmap.org)" — access.log 行 550-627
  • feroxbuster: "feroxbuster/2.2.1" — access.log 行 632-640
  • curl: "curl/7.74.0" — access.log 行 631

Q2: 哪个端点容易受到暴力破解攻击？

答案: /rest/user/login

Hydra 工具对该端点发起了大规模暴力破解攻击：

  • 时间范围: 09:16:27 - 09:16:32
  • 攻击模式: 交替使用 GET（返回 500）和 POST（返回 401 / 200）
  • 成功标志: 第三个批次的 POST 请求返回 200 状态码，响应体 831 字节

证据 — access.log:

行 208-223: GET /rest/user/login → 500 (Hydra 初始探测)
行 224-239: POST /rest/user/login → 401 (认证失败)
...
行 450: POST /rest/user/login → 200 831 (成功登录!)
行 451-495: POST /rest/user/login → 401 (继续尝试其他密码)

Q3: 哪个端点存在 SQL 注入漏洞？

答案: /rest/products/search

sqlmap 对该端点进行了全面的 SQL 注入测试，涵盖了以下数据库引擎的有效载荷：

  • MySQL / MariaDB (SLEEP(), EXTRACTVALUE())
  • PostgreSQL (PG_SLEEP())
  • Microsoft SQL Server (WAITFOR DELAY)
  • Oracle (DBMS_PIPE.RECEIVE_MESSAGE())

sqlmap 测试完成后，攻击者在浏览器中手动构造 UNION 注入，确认漏洞可利用。

证据 — access.log:

行 550: sqlmap 初始探测
行 551-627: sqlmap 注入测试载荷
行 628-629: 手动 UNION SELECT 1,2,3,4,5,6,7,8,9 FROM Users--
行 630: UNION SELECT id, email, password FROM Users--

Q4: SQL 注入使用了哪个参数？

答案: q

所有 SQL 注入载荷均通过 URL 查询参数 q 传递。

证据 — access.log 行 550:

GET /rest/products/search?q=1 HTTP/1.1 → sqlmap 初始请求
GET /rest/products/search?q=<SQL注入payload> → 所有后续请求
GET /rest/products/search?q=qwert')) UNION SELECT id, email, password, ... FROM Users-- → 数据提取

Q5: 攻击者尝试使用哪个端点来获取文件？

答案: /ftp

feroxbuster 在目录枚举中发现 /ftp 端点，该端点提供了 FTP 服务的 Web 界面，允许浏览和下载文件。

证据 — access.log:

行 640: GET /ftp HTTP/1.1 → 200 4852 (目录列表页)
行 641: GET /ftp/www-data.bak HTTP/1.1 → 403 (直接访问被拒)
行 642: GET /ftp/coupons_2013.md.bak HTTP/1.1 → 403 (直接访问被拒)

Q6: 攻击者利用网站的哪个部分抓取了用户的电子邮件地址？

答案: product reviews

在注册并登录账户后，攻击者系统地浏览了几乎全部产品的评论页面。/rest/products/{id}/reviews 端点的响应中包含了评论者的电子邮件地址。响应体越大的评论（如产品 37 返回 503 字节，产品 20 返回 405 字节）可能包含更多用户信息。

证据 — access.log:

行 21-144: 遍历 /rest/products/{1-42}/reviews
较大响应体:
- /rest/products/37/reviews → 503 bytes
- /rest/products/42/reviews → 413 bytes
- /rest/products/20/reviews → 405 bytes
- /rest/products/38/reviews → 352 bytes
- /rest/products/41/reviews → 284 bytes

Q7: 暴力破解攻击成功了吗？登录成功的时间戳是什么？

答案: Yay, 11/Apr/2021:09:16:31 +0000

Hydra 对 /rest/user/login 的 Web 暴力破解在 09:16:31 成功获取了 200 响应码（第 450 行），这表明攻击者成功登录了一个有效账户。

此外，SSH 暴力破解在 09:41:19 也成功了（auth.log 第 186 行）。

证据 — access.log:

行 450: [11/Apr/2021:09:16:31 +0000] "POST /rest/user/login HTTP/1.0" 200 831

证据 — auth.log:

行 186: Apr 11 09:41:19 thunt sshd[8260]: Accepted password for www-data from 192.168.10.5 port 40112 ssh2

Q8: 攻击者从存在 SQL 注入漏洞的端点获取了哪些用户信息？

答案: email, password

攻击者使用 UNION SELECT 注入从 Users 表中提取了三个字段：id、email 和 password。

证据 — access.log:

行 630: qwert')) UNION SELECT id, email, password, '4', '5', '6', '7', '8', '9' FROM Users--

行 631: 相同请求使用 curl/7.74.0  → 200 3742
返回了 3742 字节数据，确认成功提取了用户邮箱和密码

Q9: 他们尝试从存在漏洞的端点下载哪些文件？

答案: coupons_2013.md.bak, www-data.bak

攻击者先通过 Web 界面（/ftp/）尝试访问这些文件（返回 403），然后通过 FTP 协议直接匿名登录下载。

证据 — access.log:

行 641: GET /ftp/www-data.bak HTTP/1.1 → 403
行 642: GET /ftp/coupons_2013.md.bak HTTP/1.1 → 403

证据 — vsftpd.log:

行 37: OK DOWNLOAD: Client "::ffff:192.168.10.5", "/www-data.bak", 2602 bytes, 544.81Kbyte/sec
行 38: OK DOWNLOAD: Client "::ffff:192.168.10.5", "/coupons_2013.md.bak", 131 bytes, 3.01Kbyte/sec

Q10: 用于检索文件的服务和帐户名是什么？

答案: ftp, anonymous

攻击者通过 FTP 服务匿名登录下载文件。密码使用 IEUser@（Windows 匿名 FTP 登录的常见模式）。

证据 — vsftpd.log:

行 17: [ftp] OK LOGIN: Client "::ffff:192.168.10.5", anon password "IEUser@"
行 18: [ftp] OK LOGIN: Client "::ffff:192.168.10.5", anon password "IEUser@"
行 29: [ftp] OK LOGIN: Client "::ffff:192.168.10.5", anon password "IEUser@"

Q11: 使用什么服务和用户名获取了服务器的 shell 访问权限？

答案: ssh, www-data

攻击者通过 SSH 暴力破解成功获取了 www-data 用户的密码，建立了两个 SSH 会话，获得了 shell 访问权限。

证据 — auth.log:

行 186: Accepted password for www-data from 192.168.10.5 port 40112 ssh2
行 187: session opened for user www-data by (uid=0)
行 193: Accepted password for www-data from 192.168.10.5 port 40114 ssh2
行 194: session opened for user www-data by (uid=0)

攻击链总结

[1] Nmap 侦察 (09:08)
    ↓
[2] Web 应用探索、注册账户、登录 (09:08-09:15)
    ↓
[3] 产品评论抓取 → 收集电子邮件 (09:09-09:11)
    ↓
[4] Hydra Web 暴力破解 → 获取额外账户 (09:16)
    ↓
[5] 使用被盗账户浏览更多内容 (09:20-09:26)
    ↓
[6] sqlmap + 手动 SQL 注入 → 提取 Users 表 (email, password) (09:29-09:31)
    ↓
[7] feroxbuster 目录枚举 → 发现 /ftp (09:34)
    ↓
[8] FTP 匿名登录 → 下载 www-data.bak, coupons_2013.md.bak (09:35-09:36)
    ↓
[9] SSH 暴力破解 www-data → 获取 Shell 访问 (09:38-09:41)
    ↓
[10] sudo / su 权限提升尝试 (09:42-09:43)

影响评估

  • 数据泄露: Users 表中所有用户的 id、email、password 已泄露
  • 账户沦陷: 通过 Hydra 暴力破解和 SQL 注入获取了多个有效账户
  • Shell 访问: 攻击者通过 SSH 以 www-data 身份获得了服务器命令行访问权限
  • 敏感文件泄露: www-data.bak（2602 字节）和 coupons_2013.md.bak（131 字节）被下载
  • 权限提升尝试: 攻击者在获得 shell 后尝试了 sudo 和 su 命令，表明他们试图获取 root 权限
