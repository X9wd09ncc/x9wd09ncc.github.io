IDE

题目描述：一个能轻松提升你的计数技巧的工具！

信息收集

端口扫描

访问网站，啥也没有…

再看 62337 端口，是一个登录页面

尝试弱口令爆破密码，不行

漏洞利用

FTP 匿名登录

再尝试 ftp，发现可以匿名登录，在 ... 目录下发现文件 -

下载文件，是 drac 发给 john 的邮件，大概内容是把 john 的密码修改成了默认密码，以及关心图像文件

弱口令爆破

默认密码大概是一个弱密码，有了用户名后尝试再次爆破

成功得到密码 password

Webshell 上传

使用 john 和 password 登录，进去有四个 py 文件

后面在项目那里发现可以新建项目，并且可以上传文件，上传一个单文件 PHP shell

然后访问 /workspace/ananda/shell.php

查看 user.txt，但是没有查看权限

MySQL 密码发现

进入 drac 用户目录，列出该用户下文件，发现对 .bash_history 有可读权限

cat 查看发现 mysql 登录口令，包含账号密码

考虑到用户名都是 drac，猜测 drac 用户的密码和 mysql 的密码是相同的

反弹 Shell

由于这个是网页 shell，无法输入密码，所以需要反弹 shell 并且创建一个伪终端

利用 mkfifo + nc 反弹

本机监听，用 script 创建一个伪终端

输入 mysql 的密码，成功切换用户

权限提升

Systemd 服务提权

sudo -l 发现可以以 root 执行 vsftpd 的重启

先查看该程序的状态

它加载了 /lib/systemd/system/vsftpd.service 文件

发现有写入的权限

查看该文件

ExecStartPre 键是服务启动前置执行命令

修改 ExecStartPre=/bin/chmod +s /bin/bash

修改完后直接重启 vsftpd，然后用 bash -p 提权

总结

知识点

  • FTP 匿名登录
  • 弱口令爆破
  • Webshell 上传
  • Systemd 服务提权：修改 service 文件的 ExecStartPre
