TryHackMe_Thompson

发表于2026-05-14|更新于2026-05-14|渗透测试

Thompson

boot2root machine for FIT and bsides guatemala CTF

端口扫描

序号 Host地址 端口 协议 Uri地址 标题 Banner 状态码
1 10.146.146.159 8009 AJP13 10.146.146.159:8009 Apache Jserv i 0
2 10.146.146.159 8080 HTTP http://10.146.146.159:8080 Apache Tomcat/8.5.5 Tomcat-Login, Apache-Tomcat, Tomcat默认页面, lexmark 200
3 10.146.146.159 22 SSH 10.146.146.159:22 OpenSSH 7.2p2 Ubuntu 4ubuntu2.8 0

先访问 8080 端口，是一个Tomcat 默认页面，然后没有其他信息

接着对该地址进行目录扫描

访问 /manager，出现了登录框，尝试输入几个弱密码，三次失败后弹出错误页面

但是该错误页面却显示了正确的账号密码

使用此账号密码成功登录

在该页面下发现一个文件上的点，可以上传 war 包

利用 msfvenom 生成一个 shell.war

上传后会有一个 /shell 链接，本机监听 4444 端口，同时访问该链接

成功反弹，查找 user.txt，得到第一个 flag

来到 jack 目录下，有一个 id.sh 文件，且权限是 777

查看该文件，发现它会把 id 命令的执行结果写入 test.txt

查看 test.txt 文件，结果是 root 用户，也就是 id.sh 是以 root 权限执行的

同时注意到 test.txt 的修改时间是此时的时间，也就是说这可能是一个定时任务

在 cron 里面搜索，确定了这就是一个定时任务

由于有写入权限，那就将反弹命令写入 id.sh

同时监听写入端口，等待一会后成功拿到 root 权限
