TryHackMe - b3dr0ck Writeup

时间: 2026-06-05 02:14:19 (北京时间)

难度: Easy

靶场主题: 摩登原始人 (The Flintstones) - Fred Flintstone 和 Barney Rubble

题目描述

Barney 正在设置 ABC (Abbadabba Broadcasting Compandy) 的 Web 服务器，他尝试使用 TLS 证书来保护连接，但遇到了一些问题。我们需要帮助他解决服务器问题，并找到隐藏的 Easter eggs。

需要回答的问题：

  1. What is the barney.txt flag?
  2. What is fred’s password?
  3. What is the fred.txt flag?
  4. What is the root.txt flag?

攻击路径图

踩坑记录

1. 密码破解失败

一开始我尝试用 hashcat、john 等工具破解 Barney 的 MD5 哈希 d1ad7c0a3805955a35eb260dab4180dd，但都没有成功。后来发现这个哈希本身就是密码！

教训：有时候 CTF 的 trick 就是让你以为需要做复杂的操作，实际上答案就在眼前。

2. SSH 交互式登录

由于工具限制，无法直接进行交互式 SSH 登录。解决方法是使用 Python 的 paramiko 库来自动化 SSH 连接。

3. 证书权限问题

使用 certutil 生成的证书保存在 /usr/share/abc/certs/ 目录，但当前用户没有读取权限。不过 certutil 在输出中打印了证书和密钥的内容，可以直接使用。

4. 多层编码解码

root 密码使用了三层编码：base32 → base32 → base64。一开始我只解码了一层，得到的还是编码的字符串，让我困惑了一段时间。

解决方法：继续解码，直到得到可读的字符串。

关键知识点

  1. TLS 客户端证书认证：靶场使用客户端证书进行身份验证，我们需要获取正确的证书才能访问服务。

  2. sudo 权限滥用：

    • barney 可以运行 certutil 生成新证书
    • fred 可以运行 base32/base64 读取敏感文件

  3. 编码绕过：通过 base32/base64 命令读取受保护的文件，这是一种常见的权限绕过技术。

  4. 信息收集的重要性：端口 9009 的服务是整个攻击链的起点，如果不仔细枚举端口，可能会错过它。

工具使用

  • nmap: 端口扫描
  • netcat (nc): 连接 TCP 服务，获取 TLS 凭证
  • openssl: TLS 连接和证书操作
  • ssh: 远程登录
  • paramiko: Python SSH 库（自动化攻击）
  • base32/base64: 编码解码

总结

b3dr0ck 是一个设计精巧的 CTF 靶场，将《摩登原始人》的主题融入到技术挑战中。整个攻击链清晰：

  1. 从端口 9009 获取 TLS 凭证
  2. 使用证书获取 Barney 的密码
  3. 利用 sudo certutil 生成 Fred 的证书
  4. 使用 Fred 的证书获取 Fred 的密码
  5. 利用 sudo base32/base64 读取 root 密码
  6. 三层编码解码获取最终密码

每一步都有明确的线索，但也有一定的 trick（比如 MD5 哈希本身就是密码）。靶场涵盖了信息收集、证书认证、sudo 权限分析、编码解码等多个知识点，是一个很好的入门级 CTF。
