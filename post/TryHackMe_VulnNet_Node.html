访问 8080 端口，是一个类似于博客的网站，但是文章都看不了，只有一些用户名
左侧有一个登录按钮

但是这个登录页面是一个纯粹的静态页面，没有任何后端的交互
但是网站却给了一个 cookie，看起来像是 base64

eyJ1c2VybmFtZSI6Ikd1ZXN0IiwiaXNHdWVzdCI6dHJ1ZSwiZW5jb2RpbmciOiAidXRmLTgifQ==

解码后是一个 json 数据，有用户名和身份判断

将 cookie 修改为 admin试试，成功以admin身份登录

然后尝试访问登录页面，看看是否会跳转到其他页面，但依旧不行
猜测该网站肯定是以某个方式利用了 cookie，也就是 cookie 是一个可利用的注入点，所谓的admin身份只是其中一个利用点
现在需要弄清楚它是怎么用的，所以可以利用 sql 注入中常用的报错注入，可以通过传入一个错误的 cookie 来查看相关的报错信息

这是一个的 Node.js 反序列化漏洞 错误信息，使用了 node-serialize 这个包，在 server.js 的第16行调用了 unserialize() 函数

既然如此可以尝试注入恶意代码，例如执行 id 命令

这看起来是 typeBuffer 后面跟着一系列数字，这些数字应该是ASCII码
进行一个大概的分组，然后解码

反正无论如何都是成功执行了恶意代码，也就是存在 cookie 注入的反序列化漏洞
