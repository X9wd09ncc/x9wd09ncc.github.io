Madness

Will you be consumed by Madness?

端口扫描。只开了两个端口

序号 Host地址 端口 协议 Uri地址 标题 Banner 状态码
1 10.144.152.189 22 SSH 10.144.152.189:22 OpenSSH 7.2p2 Ubuntu 4ubuntu2.8 0
2 10.144.152.189 80 HTTP http://10.144.152.189:80 Apache2 Ubuntu Default Page: It works apache2-ubuntu默认页面,Apache2Debian默认页,Apache/2.4.18 (Ubuntu),Apache-HTTP-Server-Ubuntu,Apache2 默认页,Apache-Web-Server 200

访问网页，没什么东西

尝试目录扫描，也没有什么信息

重新回到网页，F12 看到了一条注释，该注释在一张图片下面，且该图片在网页里面并不能正常显示，可能是损坏了

利用 wget 下载图片

把图片放到 010 editor 里面，图片后缀是 jpg，但是头文件信息确实 png

因此修改成正确的头文件。打开图片有一个隐藏目录

访问 th1s_1s_h1dd3n，页面很简洁，还是 F12 又有一条注释，提示 sercet 的值在 0-99 之间

用 burpsuite 爆破，得到一个密码 y2RPJ4QaPF!B

用得到的密码来对 thm.jpg 进行隐写分析，发现里面有一个文件，将文件分离出来

查看 hidden.txt 得到用户名 wbxre

我一开始用这个用户名和密码登录 ssh，但是不行，然后就卡住了

最后看 wp 才发现在题目页面给的那张图片也可以进行隐写？？？？？只能说逆天

同样使用 steghide 进行分析，而且不需要密码

得到一个新的密码 *axA&GF8dP

这里再次尝试登录 ssh ，结果怎么死活都登不上去，只好有看一下 wp ，结果告诉我需要前面那个用户名进行 rot13 解码，解码结果为 joker……逆天

好吧，用 joker 和 *axA&GF8dP 总算是登进去了

得到第一个 flag

利用 wget 下载本地的 linpeas.sh，赋予权限并运行

把结果交给 ai 分析

Screen 4.5.0 版本存在一个著名的漏洞（CVE-2017-5618），当配合 SUID 位时，可以用来读取任意文件或提权。

searchsploit 搜索相关漏洞，并将 41154.sh 复制下来

再次利用 wget 将本地 exp.sh 上传并赋予权限

运行后直接得到 root 权限
