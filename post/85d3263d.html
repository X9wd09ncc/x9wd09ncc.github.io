IDE

题目描述：一个能轻松提升你的计数技巧的工具！

端口扫描

开放了21，22，80，还有一个 62337

先访问 80 端口，啥也没有…

再看 62337 端口，是一个登录页面

尝试弱口令爆破密码，不行

再尝试 ftp，发现可以匿名登录，ls 列出目录，在 ... 目录下发现文件 -

get 下载文件，是 drac 发给 john 的邮件，大概内容是把 john 的密码修改成了默认密码，以及关心图像文件

默认密码大概是一个弱密码，有了用户名尝试再次爆破

成功得到密码 password

使用 john 和 password 登录

进去有四个 py 文件

让 ai 分析一下有无利用点，好像没有

文件名 作用
client.py 客户端：连接服务器，发送本地摄像头画面，接收服务器画面并显示
videofeed.py 视频采集 + 图像编解码（OpenCV + PIL）
videosocket.py 自定义套接字：解决 TCP 粘包，专门发送 / 接收完整视频帧
server.py 服务器：监听端口，接收客户端画面，发送本地摄像头画面

后面在项目那里发现可以新建项目，好可以上传文件，上传一个单文件 PHP shell

然后访问 shell.php，我的路径是 /workspace/ananda/shell.php

搜索 user.txt，但是没有查看权限

进入 drac 用户目录，列出该用户下文件，发现对 .bash_history 有可读权限

cat 查看发现 mysql 登录口令，包含账号密码

尝试登录 mysql，发现没有这玩意

考虑到用户名都是 drac，猜测 drac 用户的密码和 mysql 的密码是相同的

由于这个 shell 是文件 shell，无法输入密码，所以需要反弹 shell 并且创建一个伪终端

里面有 nc，利用 nc 来反弹 shell，-e 命令没有，那就换一种方式

本机 nc 监听，用 script 创建一个伪终端

输入 drac 的账号密码，成功得到 use.txt

sudo -l 发现可以 root 执行 vsftpd 的重启

先查看该程序的状态

它加载了 /lib/systemd/system/vsftpd.service 文件

ls 加载的文件，发现有写入的权限

cat 查看该文件

ExecStartPre 键是服务启动前置执行命令

printf 修改 ExecStartPre=/bin/chmod +s /bin/bash

修改完后直接重启 vsftpd，然后用 bash -p 提权
