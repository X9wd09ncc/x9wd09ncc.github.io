Wgel CTF

Can you exfiltrate the root flag?

端口扫描

序号 Host地址 端口 协议 Uri地址 标题 Banner 状态码
1 10.146.188.205 22 SSH 10.146.188.205:22 OpenSSH 7.2p2 Ubuntu 4ubuntu2.8 0
2 10.146.188.205 80 HTTP http://10.146.188.205:80 Apache2 Ubuntu Default Page: It works Apache-HTTP-Server/2.4.18, apache2-ubuntu默认页面, Apache/2.4.18 (Ubuntu), Apache-HTTP-Server-Ubuntu, Apache2 默认页, Apache2Debian默认页, Apache-Web-Server 200

访问网站，是一个 apache 默认页面

但是在源码里面找到一条注释，目前不知道有什么用

目录扫描

访问 /sitemap

使用了 una cms，网上搜索相关漏洞，有一个 CVE-2025-32101，但是访问 /menu.php 显示 404，也就没有利用点

继续目录扫描

访问 /.ssh，有一个 id_rsa 文件，将私钥复制到本机，赋予 600 权限，用 ssh2john 转化为哈希格式，结果提示无密码

那就直接登录，用户名之前在注释里面有一个 jessie

成功登录

这个 flag 文件藏得还挺深

用户可以以 root 权限执行 /usr/bin/wget 命令

可以利用 wget 命令的文件下载功能，并结合其 --post-file 选项，来实现任意文件读取

结合之前的文件命名方式，猜测 root 的 flag 文件名是 root_flag.txt

监听端口，成功读到 flag
