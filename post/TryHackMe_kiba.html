题目概述

Kiba 是一台 Easy 难度的 Linux 渗透靶机，重点考察 Kibana CVE-2019-7609（原型污染 + RCE） 和 Linux Capabilities 提权。攻击者利用 Kibana Timelion 中的原型污染漏洞实现远程代码执行，再通过具有 cap_setuid 权限的 Python3 二进制文件提权至 root。

知识要点

概念 说明
原型污染（Prototype Pollution） JavaScript 基于原型继承的漏洞，攻击者可修改对象原型上的属性，影响所有继承该原型的对象
CVE-2019-7609 Kibana 6.6.0 及以下版本的 Timelion 组件存在原型污染，可结合 NODE_OPTIONS 实现 RCE
Linux Capabilities 将 root 权限细分为不同能力（如 cap_setuid），拥有该能力的进程可切换 UID
cap_setuid 允许进程调用 setuid() 修改有效用户 ID，可用于提权至 root

2. Kibana 漏洞利用 — CVE-2019-7609

2.1 版本识别

访问 5601 端口，在页面中确认 Kibana 版本号为 6.5.4。

2.2 漏洞原理

CVE-2019-7609 的利用过程结合了 原型污染 和 Node.js 的 NODE_OPTIONS 特性：

  1. 原型污染：通过 Timelion 输入框，污染 label.__proto__.env 对象，注入两个环境变量
  2. NODE_OPTIONS：设置 NODE_OPTIONS='--require /proc/self/environ'，让 Node.js 启动时加载 /proc/self/environ（即进程自身环境变量）
  3. 环境变量注入代码：在环境变量中注入 AAAA='require("child_process").exec("命令");process.exit()//'，其中 // 注释掉后续内容
  4. 当 Kibana 重新加载 Canvas 页面时，Node.js 会执行环境变量中的恶意代码
