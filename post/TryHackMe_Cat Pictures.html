目录扫描

访问 8080 端口，使用了 phpBB CMS，通过查看 /docs/CHANGELOG.html 找到版本为3.3.3，这是一个很新的版本，快速搜索后没有发现任何漏洞
发现可以注册用户，注册一个用户名为 admin 的帐号，在成员里面找到了管理员用户名user
起初尝试爆破密码，但是尝试几次错误密码后就要求输入验证码

里面有一个帖子，似乎是一个提示，让我们敲击神奇数字

尝试连接这些端口都被拒绝

连接之前扫描出来的 4420 端口要输入密码

到这里就卡住了，寻找 cms 漏洞也没有找到可以利用的地方，后面看 wp 才知道有种技术叫做 Port knocking（端口敲击），而提示里面也用了 knock 这个单词
端口敲击本质上是一种向攻击者隐藏特定端口的方法。我们可以定义一些规则，只有当按照特定顺序敲击其他端口时，才能打开目标端口。这里的敲击指的是发送带有 SYN 标志的 TCP 数据包。如果我们按照服务器定义的顺序敲击端口，隐藏的端口就会打开。
所以我们需要逐个尝试论坛中提到的端口，这样就能打开之前被过滤或关闭的端口
下载工具

但是我无论是使用 knock 还是 nc，都失败了，甚至重启靶机都不行，实在是没招了

按顺序敲击端口后，21 端口就可以访问了，可以匿名登录 ftp 服务，里面有一个 note.txt

下载下来，里面有 4420 端口的密码

nc 访问端口，很多命令无法执行

用户目录下有一个 runme 文件，但是在当前环境下无法执行，需要一个常规shell

因此尝试反弹 shell

kali 监听，运行 runme，但是需要密码，因此可以将这个二进制文件下载下来，利用 nc 传输到本机

本机监听

传输过来的是一个损毁的文件，经过查看是在每一个16进制数据后面添加了00，使用 xxd 转换为十六进制文本，每字节一行，取奇数行

之前在运行 runme 的时候有文字提示，因此直接搜索相关字符
please 前面有一个 rebecca，可能是密码，输入正确密码后会复制一个 ssh key

再次运行 runme，输入密码 rebecca，之后当前目录下会多出一个 id_rsa

本来想着爆破密码，结果没有密码

ssh 连接，进去之后直接就是 root 权限

进入根目录，查看 .bash_history 文件，发现一直在编辑 /opt/clean/clean.sh 文件，还有一个 .dockerenv，说明此时在一个 docker 容器里面，因此需要逃离这个环境

查看 /opt/clean/clean.sh，就是很简单的删除 /tmp 目录下的文件，

修改，添加反弹 shell

kali 监听
