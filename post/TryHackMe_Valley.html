页面分析

访问网站，一个图片网站

点击第一个按钮，进去是很多图片，有一些图片还加载不出来

点击图片可以查看图片，且有一个专门的地址 /static/1

隐藏目录

页面只显示 18 张图片，猜测有其他图片没显示，于是生成 0-100 的 dir.txt，并用 ffuf 来扫描

结果还是只有 1-18

后面看 wp，思路没错，但是居然是 00

curl 访问，给了一个新的地址 /dev1243224123123

漏洞利用

JS 代码泄露

访问该地址，是一个登录页面，尝试爆破无果

在控制台找源代码处有一个 dev.js，里面泄露了账号密码

成功进去

意思大概就是 ftp 服务也使用了这个账号密码，并且没有使用常规的 21 端口

FTP 全端口扫描

之前只扫到了两个端口，也就是说 ftp 的端口不在 top1000 里面，现在重新扫描全端口

果然得到了新的端口 37370

序号 Host地址 端口 协议 Uri地址 标题 Banner 状态码
3 10.145.191.243 37370 FTP 10.145.191.243:37370 UnKnown 0

流量分析

使用 siemDev 和 california 登录，里面有三个 pcapng 文件，下载下来

最后在 siemHTTP2.pcapng 里面找到了账号密码

uname=valleyDev&psw=ph0t0s1234

SSH 登录

使用此账号密码登录 ssh

找到 user.txt

权限提升

二进制文件分析

sudo -l 执行无果，回到 /home 目录查看其他用户

有一个 valleyAuthenticator，应该是 valley 的凭证，并且我们有对文件的执行权限

利用 strings 将其转为 txt 形式

搜索 Valley，在上面发现哈希值

e6722920bab2326f8217e4

在 https://crackstation.net/ 解密，得到密码

liberty123

用户切换

使用新得到的账号密码登录

Python 模块劫持

sudo -l 同样不行，那么查找属于用户组的文件

有一个 base64.py，内容很长

查看定时任务

nano 修改 base64.py，添加

等待一会查看 /bin/bash 权限

然后执行 bash -p 提权
