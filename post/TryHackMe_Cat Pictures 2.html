先访问80端口，里面有很多猫咪图片，点击右上角可以查看图片信息，其中一张图片有一个可以描述

note to self: strip metadata -> 提醒自己：移除元数据

将这张图片下载下来，用 exiftool 来清除元数据，然后用 exiftool 查看处理后的图片，其中第 24 行右

Title : :8080/764efa883dda1e11db47671c4a3bbd9e.txt

curl 访问，txt内容大概是在3000端口搭建了一个代码托管平台gitea，并且给了用户名和密码，然后ansible运行在1337端口，而1337端口部署了olivetin

先访问 3000 端口，登录后有一个存储库，名字是 ansible

该仓库里面有一个 flag1.txt

接下来访问 1337 端口
OliveTin 是一个开源的 Web 界面工具，核心功能是通过点击网页上的按钮，来执行预先在后台定义好的 Shell 命令，上面有 5 个脚本，而OliveTin的脚本一般都是写入yaml文件
之前8080端口的txt提示ansible是运行在OliveTin上面的，而ansible仓库里面刚好还有一个yaml文件

查看 playbook.yaml，该脚本在会执行whoami，然后调试输出该信息，最后执行echo hi

在OliveTin上执行最中间的脚本，点击右上角 log 查看运行信息
从输出输出信息来看，成功执行了 whoami，当前用户为bismut
这意味着我们可以用过修改 playbook.yaml，在里面植入恶意脚本，然后该脚本会以bismut身份执行

将 whoami 修改为反弹 shell 的命令

kali 监听 444 端口，然后在OliveTin上执行脚本

上传 linpeas.sh并运行，发现可能存在 CVE-2021-3156

该漏洞的影响的 sudo 版本 1.8.2 到 1.8.31p2 以及 1.9.0 到 1.9.5p1
查看 sudo 版本，版本是 1.8.21p2，，确实存在漏洞

下载 exp 并上传到靶机

进入目录，先用 make 编译漏洞利用程序，再运行 ./sudo-hax-me-a-sandwich 列出目标，我们的版本是1.8.21，所以应该选择0，最后执行 ./sudo-hax-me-a-sandwich 0 来进行提权
