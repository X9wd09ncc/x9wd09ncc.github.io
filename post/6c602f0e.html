TryHackMe_Creative

发表于2026-05-12|更新于2026-05-13|渗透测试

Creative

Exploit a vulnerable web application and some misconfigurations to gain root privileges.

端口扫描，有 80 和 22 端口

访问 80 端口会重定向到 creative.thm，修改 /etc/hosts 文件即可正常访问

这种重定向的一般都是有子域名的，用 ffuf 扫描

得到子域名 beta，同样添加到 /etc/hosts 里面

访问 beta.creative.thm，是一个输入框，可以提交 url 进行测试，首先就想到了 ssrf

首先输入 file://etc/passwd，结果返回 Dead

输入 https://baidu.com 也返回此页面

再尝试提交 http://creative.thm/，成功返回了主页面

同样经过测试，当提交本地地址 http://127.0.0.1/ 也会返回该页面

也就是说当输入该 web 服务器可以连接的页面是会返回响应的内容，不能则返回 Dead，并且看起来不能访问外部的网站

因此，我们可以利用此功能来获取内部 web 服务器

使用 ffuf 扫描，得到端口 1337

输入 http://127.0.0.1:1337 返回了页面，显示了根目录下的文件

因此可以借助该功能读取 web 服务器里面的文件内容

输入 http//127.0.0.1:1337/home 找到用户 saad，输入 http://127.0.0.1:1337/home/saad 进入用户目录

首先有一个 user.txt，可以获取第一个 flag

有 .bash_history 记录了用户输入的命令，但是目前用不到

还有一个 .ssh，存放了 ssh 相关的东西，由于开启了 22 端口，所以是一个可以利用的点

输入 http://127.0.0.1:1337/home/saad/.ssh 查看文件内容

在 .ssh 目录下发现了私钥 id_rsa

保存该私钥到 kali 里面，给予 600 权限

利用 ssh2john 将私钥转化成 John 能识别的哈希格式

再利用 john 破解该私钥，得到密码 sweetness

使用密码短语和密钥，就可以通过 ssh 登录

读取.bash_history，发现了该用户的账号密码

有了密码就可以查看权限 sudo

发现可以以 root 运行 /usr/bin/ping ，并且配置里面保留了 LD_PRELOAD 环境变量，这样我们就可以通过设置环境变量来运行命令

该变量用于指定进程在加载任何其他库之前要加载的库。

我们可以利用这一点，ping 以root运行，预加载我们写的恶意 so 库，就能直接提权

用 nano 编写一个恶意共享库 a.c，ctrl + o 再回车，然后 ctrl +x 退出

用 gcc 编译成动态链接库 a.so，再利用LD_PRELOAD提权

成功得到 root 权限，并获取第二个 flag
