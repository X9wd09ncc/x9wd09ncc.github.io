VulnNet: Internal

VulnNet Entertainment learns from its mistakes, and now they have something new for you…

漏洞利用

SMB 匿名访问

匿名访问 smb 服务成功，发现共享文件 share

进入 share，ls 列出文件目录，在 temp 里面发现 services.txt

NFS 挂载

2049 端口上还有 nfs 服务，尝试将其共享的网络文件挂载到本地的某个文件夹上

先用 showmount -e 列出共享的文件，有一个 /opt/conf

在本地创建一个文件夹，然后再将 /opt/conf 挂载到刚才创建的文件夹里面

Redis 密码获取

里面有一个 redis 文件夹值得关注，而靶机也开启了 redis 服务

里面有一个 redis.conf 配置文件，直接用 grep 搜索 requirepass

得到密码 B65Hx562F@ggAZ@F

Redis Flag 获取

用刚才得到的密码登录 redis，查询所有键

其中 internal flag 是我们需要的，type 命令获取数据类型，get 查看值

Rsync 凭据获取

再把其他的键的值也查看一下，其中 authlist 里面有一串 base64 编码

解码后得到 rsync 服务的密码 Hcg3HP67@TW@Bc72v

Rsync 文件下载

访问 rsync，里面有一个 files 模块，再访问该模块，有三个文件夹，将这三个文件夹下载到本地

User Flag

进入下载的文件，成功找到 user.txt，而且也大概确认了用户名 sys-internal

SSH 密钥上传

在该用户下有一个 .ssh 文件夹，但是里面没有东西

因此我们可以本地生成密钥然后上传上去，实现无密码登录

然后就可以直接登录了

权限提升

TeamCity 提权

上传 linpeas.sh，赋予权限并运行

能看到 root 起了 TeamCity 主进程

来到该文件目录，查看 TeamCity-readme.txt

大概就是说启动方式，以及开启的端口是 localhost:8111

我们无法访问这个地址，因此需要用 ssh 来进行转发

现在就可以访问了，是一个登录页面，且有一个 as a Super user 的链接可以点击

点击后需要 token 登录

那就直接在该目录下搜索，一般日志文件里面可能会有

进去后需要创建一个项目

总结

知识点

  • SMB 匿名访问
  • NFS 挂载
  • Redis 密码获取
  • rsync 文件同步
  • SSH 密钥上传实现无密码登录
  • TeamCity 提权
